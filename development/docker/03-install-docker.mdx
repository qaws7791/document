## 3.1 시스템 요구 사항 확인

Docker를 설치하기 전에, 여러분의 시스템이 Docker 실행에 필요한 최소 요구 사항을 충족하는지 확인하는 것이 중요합니다. 요구 사항은 운영체제별, Docker 버전별로 다를 수 있으므로, **항상 공식 Docker 문서를 참조**하여 최신 정보를 확인하는 것이 가장 좋습니다.

**일반적인 요구 사항:**

* **운영체제 버전:** 특정 버전 이상의 Windows, macOS, 또는 Linux 배포판이 필요합니다.
* **하드웨어 가상화 지원:** 대부분의 경우, CPU에서 하드웨어 가상화 기술(Intel VT-x 또는 AMD-V)을 지원하고 BIOS/UEFI 설정에서 활성화되어 있어야 합니다. 이는 Docker Desktop이 내부적으로 사용하는 가상화 기술(WSL 2, Hyper-V, HyperKit 등)에 필요합니다.
  * **확인 및 활성화 방법:** 시스템 BIOS/UEFI 설정 메뉴(주로 부팅 시 F2, Del, Esc 등 키 입력)에 진입하여 "Virtualization Technology", "VT-x", "AMD-V", "SVM" 등의 옵션을 찾아 "Enabled"로 설정합니다. 설정 방법은 메인보드 제조사마다 다를 수 있습니다. Windows에서는 작업 관리자(Ctrl+Shift+Esc)의 '성능' 탭에서 CPU 섹션의 '가상화' 항목 상태를 확인할 수도 있습니다.

**최신 시스템 요구 사항 확인 (공식 문서 링크):**

* **Windows용 Docker Desktop:** [https://docs.docker.com/desktop/install/windows-install/](https://docs.docker.com/desktop/install/windows-install/)
* **macOS용 Docker Desktop:** [https://docs.docker.com/desktop/install/mac-install/](https://docs.docker.com/desktop/install/mac-install/)
* **Linux용 Docker Engine:** 각 배포판별 설치 가이드 페이지에서 확인 ([https://docs.docker.com/desktop/setup/install/linux/](https://docs.docker.com/desktop/setup/install/linux/))

요구 사항을 충족하지 못하면 Docker가 제대로 설치되지 않거나 실행되지 않을 수 있으니 반드시 먼저 확인하시기 바랍니다.


## 3.2 Windows에 Docker Desktop 설치하기

Windows 환경에서는 Docker Desktop을 설치하는 것이 가장 일반적이고 편리한 방법입니다.

**사전 요구 사항:**

* 지원되는 Windows 버전 (예: Windows 10 64-bit: Pro, Enterprise, Education 버전 또는 Home 버전 / Windows 11 64-bit). 정확한 버전은 공식 문서를 확인하세요.
* WSL 2 (Windows Subsystem for Linux version 2) 기능 활성화 또는 Hyper-V 활성화 (Docker Desktop 설치 프로그램이 필요시 활성화를 도와줄 수 있습니다). **WSL 2 사용이 강력히 권장됩니다.** WSL 2는 Linux 커널을 Windows에서 직접 실행하여 더 나은 성능과 호환성을 제공합니다.

**설치 절차:**

1. **설치 파일 다운로드:** 공식 Docker 웹사이트 ([https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/)) 로 이동하여 Windows용 Docker Desktop 설치 파일 (`.exe`)을 다운로드합니다.
2. **설치 프로그램 실행:** 다운로드한 `.exe` 파일을 실행합니다.
3. **설정 확인:** 설치 과정 중 WSL 2 기능이 비활성화되어 있다면 설치 프로그램이 이를 활성화하도록 제안할 수 있습니다. 특별한 이유가 없다면 **"Use WSL 2 instead of Hyper-V" (권장)** 옵션이 선택되었는지 확인하고 진행합니다. 필요한 경우 시스템 재부팅이 요구될 수 있습니다.
4. **설치 완료 및 실행:** 설치가 완료되면 Docker Desktop을 실행합니다. 처음 실행 시 서비스 약관 동의 과정이 있을 수 있습니다.
5. **Docker Desktop 대시보드:** 성공적으로 실행되면 화면 오른쪽 하단 시스템 트레이에 고래 모양 아이콘이 나타나고, Docker Desktop 대시보드 창이 열립니다.


## 3.3 macOS에 Docker Desktop 설치하기

macOS 환경에서도 Docker Desktop을 사용하여 Docker를 쉽게 설치하고 관리할 수 있습니다.

**사전 요구 사항:**

* 지원되는 macOS 버전. (공식 문서를 통해 최신 지원 버전을 확인하세요.)
* 충분한 RAM (최소 4GB 권장) 및 디스크 공간.

**설치 절차:**

1. **설치 파일 다운로드:** 공식 Docker 웹사이트 ([https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/)) 로 이동하여 macOS용 Docker Desktop 설치 파일 (`.dmg`)을 다운로드합니다.
    * **주의:** 사용 중인 Mac의 칩 종류(Intel 또는 Apple Silicon - M1/M2/M3 등)에 맞는 설치 파일을 다운로드해야 합니다. 웹사이트에서 자동으로 감지해주거나 선택 옵션을 제공합니다.
2. **디스크 이미지 마운트:** 다운로드한 `.dmg` 파일을 더블클릭하여 디스크 이미지를 마운트합니다.
3. **애플리케이션 설치:** 열린 창에서 Docker 아이콘을 `Applications` 폴더로 드래그 앤 드롭합니다.
4. **Docker Desktop 실행:** `Applications` 폴더에서 Docker를 실행합니다.
5. **권한 부여:** 처음 실행 시 Docker가 시스템 설정을 변경하거나 네트워크 구성 등을 위해 관리자 권한을 요청할 수 있습니다. 비밀번호를 입력하여 권한을 부여합니다.
6. **Docker Desktop 대시보드:** 상단 메뉴 막대에 고래 모양 아이콘이 나타나고 Docker Desktop 대시보드 창이 열리면 설치가 성공적으로 완료된 것입니다.


## 3.4 Linux (Ubuntu/CentOS)에 Docker Engine 설치하기

Linux 환경에서는 Docker Desktop for Linux를 사용할 수도 있지만, 많은 개발자나 서버 환경에서는 GUI 없이 Docker Engine(서버 데몬 및 CLI)만 직접 설치하는 것을 선호합니다. 여기서는 Docker Engine 설치 과정을 개괄적으로 설명합니다.

**매우 중요:** Linux 배포판(Ubuntu, Debian, CentOS, Fedora 등)과 버전에 따라 설치 방법, 패키지 이름, 명령어 등이 **상당히 다를 수 있습니다.** 따라서 아래 설명은 일반적인 흐름을 이해하는 용도로만 참고하시고, **반드시 해당 Linux 배포판에 맞는 공식 Docker 설치 문서를 따라 진행**하시기 바랍니다.

**일반적인 설치 절차 (예시: Ubuntu 기준):**

1. **이전 버전 제거:** 구 버전의 Docker가 설치되어 있다면 충돌을 방지하기 위해 먼저 제거합니다.

    ```bash
    for pkg in docker.io docker-doc docker-compose docker-compose-v2 podman-docker containerd runc; do sudo apt-get remove $pkg; done
    ```

2. **Docker 리포지토리 설정:** Docker 공식 GPG 키를 추가하고 Docker의 `apt` 리포지토리를 시스템에 등록합니다. 이는 시스템 패키지 관리자가 Docker 패키지를 찾고 신뢰할 수 있도록 하기 위함입니다. (공식 문서의 정확한 명령어를 사용하세요.)

    ```bash
    sudo apt-get update
    sudo apt-get install ca-certificates curl
    sudo install -m 0755 -d /etc/apt/keyrings
    sudo curl -fsSL <https://download.docker.com/linux/ubuntu/gpg> -o /etc/apt/keyrings/docker.asc
    sudo chmod a+r /etc/apt/keyrings/docker.asc


    # Add the repository to Apt sources

    echo \
    "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
    $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}") stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    sudo apt-get update
    ```

3. **Docker Engine 설치:** 최신 버전의 Docker Engine, CLI, Containerd 등을 설치합니다. `docker-compose-plugin`을 함께 설치하면 `docker compose` 명령어를 사용할 수 있습니다.

    ```bash
    sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
    ```

4. **설치 확인:** Docker 서비스가 정상적으로 실행 중인지 확인하고, `hello-world` 이미지를 실행하여 설치를 검증합니다. (자세한 내용은 3.5 참조)

    ```bash
    sudo systemctl status docker
    sudo docker run hello-world
    ```

5. **(권장) Post-installation 단계:**
    * **sudo 없이 Docker 사용하기:** 매번 `sudo`를 입력하는 불편함을 없애기 위해 현재 사용자를 `docker` 그룹에 추가합니다. (그룹 변경 사항을 적용하려면 로그아웃 후 다시 로그인하거나 새 터미널 세션을 시작해야 할 수 있습니다.)

        ```bash
        sudo groupadd docker # 그룹이 없을 경우 생성
        sudo usermod -aG docker $USER
        newgrp docker # 현재 쉘에 즉시 적용 (선택 사항)
        docker run hello-world # sudo 없이 실행 확인(선택 사항)
        ```

    * **시스템 부팅 시 Docker 자동 시작 설정:**

        ```bash
        sudo systemctl enable docker.service
        sudo systemctl enable containerd.service
        ```

**CentOS/Fedora 등 다른 배포판:** `apt-get` 대신 `dnf` 또는 `yum`을 사용하며, 패키지 이름이나 설정 방법이 다를 수 있습니다. 반드시 해당 배포판의 공식 Docker 문서를 참조하세요.


## 3.5 설치 확인 및 기본 명령어

어떤 운영체제에 Docker를 설치했든, 다음 명령어들을 터미널(Windows에서는 PowerShell 또는 CMD, WSL 터미널)에서 실행하여 설치가 올바르게 완료되었는지 확인할 수 있습니다. (Linux에서 `docker` 그룹에 사용자를 추가하지 않았다면 명령어 앞에 `sudo`를 붙여야 할 수 있습니다.)

1. **Docker 버전 확인:** Docker 클라이언트와 서버(엔진)의 버전 정보를 확인합니다.

    ```bash
    docker version
    ```

    출력 결과에서 Client와 Server (또는 Engine) 정보가 모두 정상적으로 표시되면 기본적인 설치는 성공한 것입니다.

2. **Docker 시스템 정보 확인:** Docker 엔진의 상세한 설정 및 현재 상태(실행 중인 컨테이너 수, 이미지 수, 스토리지 드라이버, 운영체제 정보 등)를 확인합니다.

    ```bash
    docker info
    ```

3. **`hello-world` 컨테이너 실행:** Docker 설치를 확인하는 가장 확실한 방법입니다. 이 명령어를 실행하면 다음과 같은 작업이 순서대로 이루어집니다.
    * 로컬 시스템에 `hello-world` 이미지가 있는지 확인합니다.
    * 이미지가 없다면, Docker Hub (기본 레지스트리)에서 `hello-world` 이미지를 다운로드(`pull`)합니다.
    * 다운로드한 이미지를 기반으로 새로운 컨테이너를 생성하고 실행(`run`)합니다.
    * 컨테이너는 간단한 환영 메시지를 출력하고 종료됩니다.

    ```bash
    docker run hello-world
    ```

    터미널에 "Hello from Docker!" 로 시작하는 메시지가 성공적으로 출력된다면, 여러분의 시스템에서 Docker 엔진이 이미지를 가져와 컨테이너를 실행할 수 있다는 의미이므로 설치가 성공적으로 완료된 것입니다. 출력된 메시지에는 이 과정에 대한 설명도 포함되어 있으니 한번 읽어보시는 것도 좋습니다.


## 3.6 Docker Desktop 설정 살펴보기 (Windows/macOS 사용자)

Windows 또는 macOS에 Docker Desktop을 설치한 사용자라면, 시스템 트레이(Windows)나 메뉴 막대(macOS)의 Docker 아이콘을 클릭하여 나타나는 메뉴에서 톱니바퀴 모양의 **Settings (설정)** 아이콘을 통해 다양한 설정을 변경할 수 있습니다.

주요 설정 항목들은 다음과 같습니다 (버전에 따라 다를 수 있음):

* **General (일반):** Docker Desktop 자동 시작 여부, 테마 설정 등을 변경할 수 있습니다.
* **Resources (리소스):** Docker가 사용할 수 있는 CPU 코어 수, 메모리(RAM) 크기, 디스크 이미지 크기 등을 조절할 수 있습니다. Windows에서는 WSL 통합 관련 설정을 여기서 관리합니다. (초기에는 기본 설정을 유지하는 것이 좋습니다.)
* **Docker Engine:** Docker 데몬(`dockerd`)의 설정을 JSON 형식으로 직접 수정할 수 있는 고급 설정 메뉴입니다.
* **Kubernetes:** Docker Desktop에 내장된 Kubernetes 클러스터를 활성화할 수 있습니다. (이 문서의 범위를 벗어납니다.)
* **Extensions (확장 기능):** Docker Desktop의 기능을 확장하는 다양한 서드파티 도구들을 설치하고 관리할 수 있습니다.
