# 보안 애플리케이션


## 사용자 입력 검증

크로스 사이트 스크립팅(XSS), SQL 인젝션 등

- 신뢰할 수 있는 모든 입력을 수용하기 위해서는 데이터를 **검증**해야 하고 표시할 때는 적절히 이스케이프 해야 한다
- 사용자 입력을 검증하는 필터를 작성하기 위해서는 블랙리스트 필터 기반 ❌ -> 화이트리스트 필터 기반✅으로 작성하여 알려지지 않은 위험을 방지할 수 있다
- 예시: form input 데이터, URL 매개변수, 타 사이트 간 메시지, 로컬 스토리지, 쿠키 등


## 크로스 사이트 요청 위조(CSRF)

- 부작용이 발생할 수 있는 가능성이 있는 사용자의 요청에 대해서 진짜 사용자가 의도한 것인지, 다른 악의적인 사용자가 속여 발생한 요청인지 확인해야 한다
- 숨겨진 값을 form에 주입하여 서버에서 값을 비교하여 진짜 사용자의 요청인지 확인할 수 있다
- 서버에서 요청의 Origin 헤더 값을 통해 어디서 온 요청인지 확인할 수 있다


## 클랙재킹

- 사용자의 인터페이스를 속여 행동을 가로채서 다른 원치 않는 작업을 수행되는 가능성이 없도록 설계해야 한다
- `iframe`를 화면에 겹치도록 심어 `iframe` 내의 다른 사이트의 클릭 유도하도록 행위를 방지해야 한다
