## 1.1. 리눅스 소개


### 1.1.1. 리눅스란 무엇인가?

리눅스는 리누스 토르발스(Linus Torvalds)라는 핀란드 대학생이 1991년에 개발한 유닉스(UNIX) 기반의 오픈소스 운영체제입니다. 운영체제는 컴퓨터의 하드웨어 자원(CPU, 메모리, 저장 장치 등)을 관리하고, 응용 프로그램이 하드웨어를 사용할 수 있도록 중개하는 역할을 합니다.

**주요 특징:**

* **오픈소스:** 리눅스의 핵심인 커널(kernel)을 포함한 대부분의 구성 요소가 오픈소스 라이선스하에 배포됩니다. 이는 누구나 소스 코드를 자유롭게 보고, 수정하고, 배포할 수 있음을 의미합니다.
* **다양성:** 다양한 하드웨어 플랫폼(데스크탑, 서버, 임베디드 시스템 등)에서 동작하며, 용도에 따라 특화된 여러 종류의 배포판(Distribution)이 존재합니다.
* **안정성 및 보안성:** 오랜 기간 동안 많은 개발자들의 참여와 검증을 거쳐 높은 안정성과 보안성을 자랑합니다.
* **명령행 인터페이스 (CLI):** 강력한 명령행 인터페이스를 통해 시스템을 세밀하게 제어할 수 있습니다. 그래픽 사용자 인터페이스(GUI)도 지원하지만, 서버 환경에서는 주로 CLI가 사용됩니다.
* **멀티태스킹 및 멀티유저:** 여러 개의 작업을 동시에 처리할 수 있는 멀티태스킹과 여러 명의 사용자가 동시에 시스템을 사용할 수 있는 멀티유저 기능을 지원합니다.


### 1.1.2. 리눅스 철학 - 오픈소스

리눅스의 핵심 철학은 오픈소스입니다. 오픈소스는 단순히 소스 코드를 공개하는 것을 넘어 다음과 같은 가치를 지향합니다.

* **자유로운 사용:** 누구나 제약 없이 리눅스를 사용하고 배포할 수 있습니다.
* **자유로운 배포:** 리눅스를 다른 사람에게 자유롭게 복사하고 배포할 수 있습니다.
* **자유로운 수정:** 필요에 따라 소스 코드를 수정하고 개선할 수 있습니다.
* **자유로운 배포된 버전:** 수정된 버전을 다시 배포할 수 있습니다.

이러한 오픈소스 철학은 리눅스의 빠른 발전과 폭넓은 활용에 크게 기여했습니다. 전 세계의 개발자들이 자발적으로 참여하여 버그를 수정하고 새로운 기능을 추가하며, 사용자들은 이러한 혜택을 누릴 수 있습니다.


### 1.1.3. 주요 리눅스 배포판 소개

리눅스 커널을 기반으로 다양한 소프트웨어를 묶어 사용자가 편리하게 설치하고 사용할 수 있도록 만든 것이 리눅스 배포판입니다. 수많은 배포판이 존재하며, 각각의 특징과 목적에 따라 다양한 분야에서 활용됩니다.

**주요 배포판**

* **Ubuntu:** 사용자 친화적인 인터페이스와 풍부한 소프트웨어 저장소를 제공하여 초보자에게 인기가 많습니다. 데스크탑 환경뿐만 아니라 서버 환경에서도 널리 사용됩니다.
* **CentOS / RHEL (Red Hat Enterprise Linux):** 기업 환경에서 높은 안정성과 보안성을 요구하는 서버 운영체제로 널리 사용됩니다. CentOS는 RHEL의 소스 코드를 기반으로 무료로 제공되는 배포판입니다.
* **Debian:** 안정성을 최우선으로 하는 커뮤니티 주도의 배포판입니다. 폭넓은 하드웨어 지원과 방대한 소프트웨어 저장소를 자랑합니다.


### 1.1.4. 왜  리눅스를 사용하는가?

리눅스 운영체제를 사용하는 데에는 다음과 같은 여러 가지 이유가 있습니다.

* **비용 효율성:** 대부분의 리눅스 배포판은 무료로 사용할 수 있어 소프트웨어 라이선스 비용을 절감할 수 있습니다.
* **높은 안정성 및 성능:** 서버 환경에서 중요한 요소인 안정성과 성능이 뛰어납니다. 시스템 다운타임 최소화 및 효율적인 자원 활용이 가능합니다.
* **뛰어난 보안성:** 오픈소스 특성상 보안 취약점이 빠르게 발견되고 수정되며, 다양한 보안 도구 및 설정 옵션을 제공하여 강력한 보안 환경을 구축할 수 있습니다.
* **유연성 및 확장성:** 다양한 하드웨어 및 소프트웨어 환경에 유연하게 적용 가능하며, 필요에 따라 시스템을 확장하거나 맞춤 설정하기 용이합니다.
* **개발 및 운영 환경의 표준화:** 리눅스를 표준 운영체제로 사용함으로써 개발 환경과 운영 환경을 일관성 있게 유지하고 관리 효율성을 높일 수 있습니다.


### 1.1.5. 윈도우/macOS 사용자를 위한 리눅스와의 차이점 비교

윈도우나 macOS 환경에 익숙한 사용자는 리눅스를 처음 접할 때 몇 가지 차이점을 느낄 수 있습니다. 주요 차이점은 다음과 같습니다.

| 특징             | 윈도우 (Windows)                               | macOS                                          | 리눅스 (Linux)                                      |
| ---------------- | ---------------------------------------------- | ---------------------------------------------- | --------------------------------------------------- |
| **커널** | Windows NT 커널                               | XNU 커널 (BSD 기반)                              | 리눅스 커널                                         |
| **기반** | 주로 Microsoft                                 | Apple                                            | 다양한 오픈소스 커뮤니티 및 기업                     |
| **배포 방식** | 단일 운영체제 이미지                             | 단일 운영체제 이미지                             | 다양한 배포판 존재 (Ubuntu, CentOS 등)              |
| **사용자 인터페이스** | GUI (그래픽 사용자 인터페이스) 중심             | GUI (Aqua) 중심                                | GUI (GNOME, KDE 등) 및 CLI (명령행 인터페이스) 지원 |
| **소프트웨어 관리** | .exe, .msi 등의 설치 파일, Microsoft Store      | .dmg, .pkg 등의 설치 파일, App Store           | 패키지 관리 시스템 (apt, yum 등) 활용                 |
| **파일 시스템** | NTFS, FAT32                                    | APFS, HFS+                                     | ext4, XFS 등 다양한 파일 시스템 지원                 |
| **명령행 도구** | PowerShell, Command Prompt                     | Terminal (Bash, Zsh)                             | Terminal (Bash, Zsh 등) - 강력한 기능 제공         |
| **오픈소스** | 일부 오픈소스 프로젝트 존재                       | BSD 기반의 오픈소스 요소 포함                     | 핵심 철학이 오픈소스                                |
| **하드웨어 지원** | 넓은 호환성                                     | Apple 제품에 최적화                             | 다양한 하드웨어 지원, 커널 업데이트 통해 지속적 확장 |

리눅스는 명령행 인터페이스를 통해 시스템을 제어하는 방식이 윈도우나 macOS와 가장 큰 차이점 중 하나입니다. 처음에는 다소 어렵게 느껴질 수 있지만, 익숙해지면 더욱 강력하고 효율적인 시스템 관리가 가능합니다.


## 1.2. 리눅스 설치 및 환경 준비


### 1.2.1. 설치 방법 개요

리눅스를 사용하기 위해서는 먼저 시스템에 리눅스를 설치하는 과정이 필요합니다. 설치 방법은 사용 목적, 현재 사용하고 있는 운영체제, 그리고 개인적인 선호도에 따라 다양하게 선택할 수 있습니다. 주요 설치 방법은 다음과 같습니다.

* **가상 머신(Virtual Machine) 활용:** 현재 사용하고 있는 운영체제(Windows, macOS 등) 내에 가상의 컴퓨터 환경을 만들고, 그 가상 환경에 리눅스를 설치하는 방식입니다. 기존 운영체제를 유지하면서 리눅스를 학습하거나 테스트하는 데 유용합니다.
* **듀얼 부팅(Dual Booting):** 하나의 물리적인 컴퓨터에 두 개 이상의 운영체제(예: Windows와 Linux)를 설치하고, 부팅 시 어떤 운영체제를 사용할지 선택하는 방식입니다. 리눅스를 주 운영체제로 사용하거나, 필요에 따라 다른 운영체제로 전환하고자 할 때 적합합니다.
* **USB/DVD를 이용한 부팅 및 설치:** 리눅스 설치 이미지를 담은 USB 드라이브나 DVD를 이용하여 컴퓨터를 부팅하고, 하드 디스크에 리눅스를 새로 설치하는 방식입니다. 기존 운영체제를 완전히 대체하거나, 새로운 시스템에 리눅스를 설치할 때 사용됩니다.
* **WSL (Windows Subsystem for Linux):** 윈도우 환경에서 리눅스 환경을 직접 실행할 수 있도록 Microsoft에서 제공하는 기능입니다. 별도의 가상 머신 설치 없이 윈도우 내에서 리눅스 명령어를 사용하거나 리눅스 기반 도구를 실행할 수 있습니다. 개발 환경 구축에 유용합니다.
* **클라우드 환경 이용:** Amazon Web Services (AWS), Google Cloud Platform (GCP), Microsoft Azure 등의 클라우드 플랫폼에서 제공하는 리눅스 기반 가상 서버를 활용하는 방식입니다. 서버 환경 구축 및 관리에 주로 사용됩니다.


### 1.2.2. 가상 머신(Virtual Machine) 설치 가이드

가상 머신은 현재 사용 중인 운영체제 내에 독립적인 가상 컴퓨터를 만들어 리눅스를 설치하고 실행할 수 있도록 해줍니다. 대표적인 가상 머신 소프트웨어로는 VirtualBox (무료)와 VMware Workstation Player (개인 사용자 무료) 등이 있습니다.

**VirtualBox 설치 가이드:**

1. **VirtualBox 다운로드:** [Oracle VirtualBox 공식 웹사이트](https://www.virtualbox.org/)에 접속하여 사용하고 있는 운영체제에 맞는 설치 파일을 다운로드합니다.
2. **VirtualBox 설치:** 다운로드한 설치 파일을 실행하고, 화면의 안내에 따라 VirtualBox를 설치합니다. 설치 과정에서 네트워크 설정 관련 경고 메시지가 나타날 수 있으며, 특별한 이유가 없다면 기본 설정으로 진행합니다.
3. **리눅스 배포판 이미지 다운로드:** 사용하고자 하는 리눅스 배포판(예: Ubuntu)의 공식 웹사이트에서 ISO 형태의 설치 이미지를 다운로드합니다.
4. **새로운 가상 머신 생성:** VirtualBox를 실행하고, "새로 만들기" 버튼을 클릭합니다.
5. **가상 머신 설정:**
    * **이름:** 생성할 가상 머신의 이름을 입력합니다 (예: Ubuntu 22.04).
    * **머신 폴더:** 가상 머신 관련 파일이 저장될 경로를 선택합니다.
    * **종류:** "Linux"를 선택합니다.
    * **버전:** 설치할 리눅스 배포판에 맞는 버전을 선택합니다 (예: Ubuntu (64-bit)).
    * **다음** 버튼을 클릭합니다.
6. **메모리 크기 할당:** 가상 머신에 할당할 메모리 크기를 설정합니다. 호스트 PC의 총 메모리 용량을 고려하여 적절한 크기를 지정합니다 (일반적으로 2GB ~ 4GB 권장).
7. **하드 디스크 설정:** "지금 가상 하드 디스크 만들기"를 선택하고 "만들기" 버튼을 클릭합니다.
8. **가상 하드 디스크 파일 종류:** "VDI (VirtualBox Disk Image)"를 선택하고 "다음" 버튼을 클릭합니다.
9. **저장 방식:** "동적 할당" 또는 "고정 크기"를 선택합니다. "동적 할당"은 필요에 따라 용량을 늘려가는 방식이며, "고정 크기"는 처음 설정한 용량을 미리 확보하는 방식입니다. 초보자는 "동적 할당"을 권장합니다.
10. **가상 하드 디스크 크기:** 가상 머신에 할당할 하드 디스크 크기를 설정합니다 (일반적으로 20GB 이상 권장).
11. **만들기** 버튼을 클릭하여 가상 머신 생성을 완료합니다.
12. **설정 변경:** 생성된 가상 머신을 선택하고 "설정" 버튼을 클릭합니다.
13. **저장소 설정:** "저장소" 탭으로 이동하여 "비어 있음"으로 표시된 CD/DVD 아이콘을 선택하고, 오른쪽의 CD 모양 아이콘을 클릭하여 다운로드한 리눅스 ISO 이미지를 선택합니다.
14. **네트워크 설정 (선택 사항):** 필요에 따라 "네트워크" 탭에서 네트워크 연결 방식을 설정합니다 (일반적으로 "NAT" 또는 "브리지 네트워크"를 사용합니다).
15. **확인** 버튼을 클릭하여 설정을 저장합니다.
16. **가상 머신 실행 및 리눅스 설치:** 생성된 가상 머신을 선택하고 "시작" 버튼을 클릭하면 가상 머신이 부팅되고 리눅스 설치 과정이 시작됩니다. 화면의 안내에 따라 리눅스를 설치합니다.

**VMware Workstation Player 설치 가이드:**

(VirtualBox와 유사한 과정으로, VMware Workstation Player 공식 웹사이트에서 다운로드 및 설치 후, 새로운 가상 머신을 생성하고 리눅스 ISO 이미지를 연결하여 설치합니다. 상세 과정은 생략합니다.)


### 1.2.3. Windows Subsystem for Linux (WSL) 설치 및 사용법

WSL은 윈도우 환경에서 별도의 가상 머신 없이 리눅스 환경을 사용할 수 있도록 해주는 편리한 기능입니다.

**WSL 설치 방법 (Windows 10 이상):**

1. **PowerShell 관리자 권한 실행:** 시작 메뉴에서 "PowerShell"을 검색하고, "관리자 권한으로 실행"을 클릭합니다.
2. **WSL 기능 활성화:** PowerShell 창에 다음 명령어를 입력하고 실행합니다.

    ```powershell
    wsl --install
    ```

    이 명령어는 WSL의 최신 버전을 활성화하고, 기본 Ubuntu 배포판을 설치합니다.
3. **시스템 재부팅:** 설치가 완료되면 시스템을 재부팅합니다.
4. **Ubuntu 설정:** 재부팅 후 Ubuntu 콘솔 창이 자동으로 열립니다. 새로운 Linux 사용자 이름과 비밀번호를 설정합니다. 이 계정은 WSL 환경 내에서 사용되는 계정입니다.

**다른 리눅스 배포판 설치 (선택 사항):**

기본 Ubuntu 외에 다른 리눅스 배포판을 설치하고 싶다면 Microsoft Store를 이용하거나, 명령어를 통해 설치할 수 있습니다.

* **Microsoft Store 이용:** Microsoft Store에서 원하는 리눅스 배포판(예: Debian, Kali Linux 등)을 검색하여 설치합니다.
* **명령어 이용:** PowerShell에서 다음 명령어를 사용하여 설치 가능한 배포판 목록을 확인하고, 원하는 배포판을 설치할 수 있습니다.

    ```powershell
    wsl --list --online
    wsl --install <배포판 이름>
    ```

    (예: `wsl --install Debian`)

**WSL 사용법:**

* **터미널 실행:** 시작 메뉴에서 설치한 리눅스 배포판 이름(예: Ubuntu)을 검색하여 터미널을 실행합니다.
* **명령어 사용:** 일반적인 리눅스 명령어를 터미널에서 사용할 수 있습니다 (예: `ls`, `cd`, `apt update` 등).
* **윈도우 파일 접근:** WSL 환경 내에서 `/mnt/c/` 경로를 통해 윈도우의 C 드라이브에 접근할 수 있습니다. 다른 드라이브도 `/mnt/<드라이브 문자>/` 형태로 접근 가능합니다.
* **윈도우 애플리케이션 실행:** WSL 터미널에서 윈도우 실행 파일(.exe)을 직접 실행할 수도 있습니다 (경로를 정확히 지정해야 할 수 있습니다).


### 1.2.4. 표준 개발/운영 환경 접속 가이드

리눅스 기반의 개발 및 운영 환경에 접속하기 위해 **SSH (Secure Shell)** 프로토콜을 사용할 수 있습니다. SSH는 네트워크를 통해 다른 컴퓨터에 안전하게 접속하여 명령을 실행하거나 파일을 전송할 수 있도록 해주는 암호화된 통신 프로토콜입니다.

**SSH 접속 방법:**

1. **SSH 클라이언트 설치:** 윈도우 환경에서는 PuTTY, MobaXterm 등의 SSH 클라이언트 프로그램을 설치하거나, Windows 10 이상 버전에서는 내장된 OpenSSH 클라이언트를 사용할 수 있습니다 (PowerShell 또는 명령 프롬프트에서 `ssh` 명령 사용). macOS 및 리눅스 환경에서는 기본적으로 SSH 클라이언트가 설치되어 있습니다.
2. **접속 정보 확인:** 접속하고자 하는 리눅스 서버의 IP 주소 (또는 호스트 이름), 접속 포트 (기본값: 22), 그리고 사용자 계정 정보를 확인합니다. 이 정보는 시스템 관리자로부터 제공받을 수 있습니다.
3. **SSH 클라이언트 실행:** 설치한 SSH 클라이언트 프로그램을 실행하거나, 터미널 (macOS/Linux) 또는 PowerShell/명령 프롬프트 (Windows 10 이상)를 엽니다.
4. **접속 명령 실행 (터미널/PowerShell):** 터미널 또는 PowerShell 창에 다음과 같은 형식으로 SSH 접속 명령어를 입력하고 실행합니다.

    ```bash
    ssh <사용자 계정>@<IP 주소 또는 호스트 이름> -p <포트 번호>
    ```

    예시: `ssh user1@192.168.1.100 -p 22`

    포트 번호가 기본값(22)인 경우에는 `-p <포트 번호>` 옵션을 생략할 수 있습니다.

    ```bash
    ssh user1@192.168.1.100
    ```

5. **비밀번호 입력:** 서버에 처음 접속하는 경우 호스트 키에 대한 확인 메시지가 나타날 수 있습니다. "yes"를 입력하여 계속 진행합니다. 그 후, 해당 사용자 계정의 비밀번호를 입력합니다. 비밀번호를 입력할 때 화면에 아무것도 표시되지 않는 것이 일반적입니다. 비밀번호를 정확히 입력하고 Enter 키를 누릅니다.
6. **접속 완료:** 비밀번호가 올바르게 입력되면 리눅스 서버에 SSH로 접속됩니다. 명령 프롬프트가 서버의 정보로 바뀌는 것을 확인할 수 있습니다.

**PuTTY 사용법 (Windows):**

1. **PuTTY 실행:** 설치된 PuTTY 프로그램을 실행합니다.
2. **Session 설정:**
    * **Host Name (or IP address):** 접속할 서버의 IP 주소 또는 호스트 이름을 입력합니다.
    * **Port:** 접속 포트 번호를 입력합니다 (기본값: 22).
    * **Connection type:** "SSH"를 선택합니다.
3. **Open 버튼 클릭:** "Open" 버튼을 클릭하면 SSH 연결이 시도됩니다.
4. **보안 경고 (처음 접속 시):** 처음 접속하는 서버인 경우 보안 경고 창이 나타날 수 있습니다. "Yes"를 클릭하여 호스트 키를 저장하고 연결을 계속합니다.
5. **로그인:** PuTTY 터미널 창이 열리면 "login as:" 프롬프트에 사용자 계정을 입력하고 Enter 키를 누릅니다.
6. **비밀번호 입력:** "password:" 프롬프트에 비밀번호를 입력하고 Enter 키를 누릅니다. 비밀번호는 화면에 표시되지 않습니다.
7. **접속 완료:** 비밀번호가 올바르게 입력되면 리눅스 서버에 SSH로 접속됩니다.

**SSH 키 기반 인증 (권장):**

비밀번호 대신 SSH 키 쌍(공개 키와 개인 키)을 사용하여 더욱 안전하고 편리하게 서버에 접속하는 방식입니다. 키 기반 인증 설정 방법은 별도의 문서에서 자세히 다루도록 하겠습니다.


## 1.3. 첫걸음: 기본 인터페이스 사용법

리눅스 환경에 성공적으로 접속했다면, 이제 기본적인 인터페이스 사용법을 익혀야 합니다. 리눅스는 주로 명령행 인터페이스(CLI)인 터미널을 통해 상호작용하지만, 데스크탑 환경이 설치된 경우에는 그래픽 사용자 인터페이스(GUI)도 사용할 수 있습니다.


### 1.3.1. 데스크탑 환경 소개

리눅스는 다양한 데스크탑 환경을 제공합니다. 대표적인 데스크탑 환경으로는 GNOME, KDE Plasma, XFCE 등이 있습니다. 회사 표준 배포판에 기본적으로 설치된 데스크탑 환경이 있다면 해당 환경에 대한 소개를 중심으로 설명합니다.

**(데스크탑 환경 이름)의 주요 구성 요소:**

* **바탕화면 (Desktop):** 아이콘, 바로가기 등을 배치하고 작업 공간을 관리하는 기본 화면입니다.
* **패널 (Panel):** 화면 상단 또는 하단에 위치하며, 메뉴, 실행 중인 프로그램 목록, 시스템 트레이 (시계, 네트워크 상태 등)를 포함합니다.
* **메뉴 (Menu):** 설치된 애플리케이션 목록을 확인하고 실행할 수 있습니다. 일반적으로 화면 좌측 상단 또는 패널의 특정 아이콘을 통해 접근할 수 있습니다.
* **창 (Window):** 실행 중인 각 애플리케이션은 독립적인 창으로 표시됩니다. 창 제목 표시줄, 최소화/최대화/닫기 버튼 등을 제공합니다.
* **파일 관리자 (File Manager):** 파일 및 폴더를 탐색하고 관리할 수 있는 GUI 도구입니다 (예: Nautilus - GNOME, Dolphin - KDE).
* **작업 공간 (Workspace / Virtual Desktop):** 여러 개의 가상 화면을 사용하여 창들을 분리하고 작업 공간을 효율적으로 관리할 수 있습니다.

**(데스크탑 환경 사용법 간단 안내)**

* **프로그램 실행:** 메뉴에서 원하는 애플리케이션을 찾아 클릭하거나, 바탕화면의 아이콘을 더블 클릭하여 실행합니다.
* **창 관리:** 창 제목 표시줄을 드래그하여 창 위치를 이동하고, 최소화/최대화/닫기 버튼을 사용하여 창 상태를 변경합니다.
* **파일 관리:** 파일 관리자를 실행하여 파일 및 폴더를 생성, 복사, 이동, 삭제하는 등의 작업을 수행합니다.
* **작업 공간 전환:** 일반적으로 키보드 단축키 (예: Ctrl + Alt + 화살표 키) 또는 화면의 작업 공간 전환 인터페이스를 통해 작업 공간을 변경할 수 있습니다.
* **시스템 설정:** 패널의 시스템 트레이 또는 메뉴에서 시스템 설정 (네트워크, 디스플레이, 사용자 계정 등)에 접근하여 환경을 구성할 수 있습니다.

**(데스크탑 환경이 없는 서버 환경의 경우)**

서버 환경에서는 일반적으로 GUI 데스크탑 환경이 설치되어 있지 않으며, 순수하게 명령행 인터페이스(CLI)만 제공됩니다. 이 경우, 다음 섹션에서 설명하는 터미널 사용법을 익히는 것이 중요합니다.


### 1.3.2. 터미널(Terminal) / 콘솔(Console) 열기 및 기본 사용법

터미널 또는 콘솔은 리눅스 시스템과 상호작용하는 가장 기본적인 방법 중 하나입니다. 명령어를 입력하여 시스템을 제어하고 다양한 작업을 수행할 수 있습니다.

**터미널 열기:**

* **GUI 환경:** 데스크탑 환경에서는 메뉴에서 "터미널", "콘솔", "명령 프롬프트" 등의 이름으로 검색하여 실행할 수 있습니다. 일반적으로 자주 사용하는 애플리케이션 목록이나 시스템 도구 모음에 위치합니다.
* **원격 접속 (SSH):** SSH 클라이언트를 통해 서버에 접속하면 터미널 환경이 제공됩니다.
* **가상 콘솔:** GUI 환경이 실행 중인 상태에서 `Ctrl` + `Alt` + `F1` ~ `F6` 키를 누르면 텍스트 기반의 가상 콘솔로 전환할 수 있습니다. GUI 환경으로 돌아가려면 `Ctrl` + `Alt` + `F7` (또는 다른 F 키)를 누릅니다.

**기본 사용법:**

터미널을 열면 명령 프롬프트가 표시됩니다. 명령 프롬프트는 일반적으로 사용자 이름, 호스트 이름, 현재 작업 디렉토리 등을 나타냅니다. 예를 들어 다음과 같은 형태를 가질 수 있습니다.

```bash
[user@hostname ~]$
```

* `user`: 현재 로그인한 사용자 이름
* `hostname`: 현재 시스템의 호스트 이름
* `~`: 현재 작업 디렉토리 (여기서는 홈 디렉토리를 의미)
* `$`: 일반 사용자 프롬프트 (`#`은 root 사용자 프롬프트)

터미널에서는 명령어를 입력하고 Enter 키를 눌러 실행합니다. 명령어는 일반적으로 다음과 같은 구조를 가집니다.

```bash
명령어 [옵션] [인수]
```

* **명령어:** 수행하고자 하는 작업의 이름 (예: `ls`, `cd`, `pwd`)
* **옵션:** 명령어의 동작 방식을 변경하거나 추가 기능을 제공하는 부분 (일반적으로 `-` 또는 `--`로 시작)
* **인수:** 명령어의 대상이 되는 파일, 디렉토리, 사용자 이름 등

**자주 사용되는 기본 명령어:**

* `pwd` (Print Working Directory): 현재 작업 디렉토리의 경로를 출력합니다.
* `ls` (List): 현재 디렉토리의 파일 및 디렉토리 목록을 출력합니다.
  * `ls -l`: 파일의 상세 정보 (권한, 소유자, 크기, 수정 시간 등)를 함께 출력합니다.
  * `ls -a`: 숨겨진 파일 및 디렉토리 (이름이 `.`으로 시작하는 파일)를 포함하여 모두 출력합니다.
  * `ls -h`: 파일 크기를 사람이 읽기 쉬운 형태로 표시합니다 (예: 1K, 234M, 2G).
  * `ls -al`: `-a`와 `-l` 옵션을 함께 사용합니다.
* `cd` (Change Directory): 작업 디렉토리를 변경합니다.
  * `cd <디렉토리 경로>`: 지정된 경로로 이동합니다.
  * `cd`: 홈 디렉토리로 이동합니다.
  * `cd ..`: 상위 디렉토리로 이동합니다.
  * `cd -`: 이전 작업 디렉토리로 이동합니다.
* `mkdir` (Make Directory): 새로운 디렉토리를 생성합니다.
  * `mkdir <디렉토리 이름>`
* `rmdir` (Remove Directory): 비어 있는 디렉토리를 삭제합니다.
  * `rmdir <디렉토리 이름>`
* `touch` (Change file timestamps / Create an empty file): 파일의 접근 및 수정 시간을 변경하거나, 내용이 없는 빈 파일을 생성합니다.
  * `touch <파일 이름>`

**명령어 자동 완성 (Tab 키):**

터미널에서 명령어 또는 파일/디렉토리 이름을 입력하는 도중 `Tab` 키를 누르면 자동 완성을 시도합니다. 일치하는 항목이 하나인 경우 자동으로 완성되며, 여러 개인 경우 가능한 목록을 보여줍니다. 오타를 줄이고 빠르게 명령어를 입력하는 데 유용합니다.

**명령어 기록 (History):**

이전에 입력했던 명령어는 기록으로 남아있습니다. 위쪽 화살표 키 (`↑`)를 누르면 이전 명령어를 순서대로 불러올 수 있으며, 아래쪽 화살표 키 (`↓`)를 누르면 다음 명령어를 불러올 수 있습니다. `history` 명령어를 입력하면 전체 명령어 기록을 확인할 수 있습니다.


### 1.3.3. 첫 로그인 및 암호 변경

새로운 리눅스 시스템에 처음 로그인하거나, 계정이 생성된 후 처음 접속하는 경우 초기 설정을 완료해야 할 수 있습니다. 또한, 보안을 위해 주기적으로 비밀번호를 변경하는 것이 좋습니다.

**첫 로그인:**

* 가상 머신 또는 새로 설치한 시스템의 경우, 설치 과정에서 설정한 사용자 이름과 비밀번호를 사용하여 로그인합니다.
* 시스템 관리자가 계정을 생성해 준 경우에는 제공받은 사용자 이름과 초기 비밀번호를 사용하여 로그인합니다.

**암호 변경:**

로그인 후에는 보안을 위해 초기 비밀번호를 변경하는 것이 좋습니다. 터미널에서 `passwd` 명령어를 사용하여 비밀번호를 변경할 수 있습니다.

1. 터미널을 엽니다.
2. `passwd` 명령어를 입력하고 Enter 키를 누릅니다.

    ```bash
    passwd
    ```

3. 현재 비밀번호를 입력하라는 메시지가 나타나면 현재 사용하고 있는 비밀번호를 입력하고 Enter 키를 누릅니다 (화면에 표시되지 않습니다).
4. 새로운 비밀번호를 입력하라는 메시지가 나타나면 사용할 새로운 비밀번호를 입력하고 Enter 키를 누릅니다 (화면에 표시되지 않습니다).
5. 새로운 비밀번호를 다시 한번 입력하라는 메시지가 나타나면 동일한 비밀번호를 다시 입력하고 Enter 키를 누릅니다.

비밀번호가 성공적으로 변경되면 메시지가 표시됩니다. 안전한 비밀번호를 설정하고 주기적으로 변경하는 것이 중요합니다. 비밀번호는 다른 사람이 쉽게 추측할 수 없는 복잡한 조합 (영문 대소문자, 숫자, 특수문자 포함)으로 설정하는 것이 좋습니다.
