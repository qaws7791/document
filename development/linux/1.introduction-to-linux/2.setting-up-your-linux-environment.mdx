Chapter 1에서 운영체제가 무엇인지, 그리고 리눅스가 왜 매력적인 선택인지 함께 알아보았습니다. 이론적인 배경을 이해하는 것도 중요하지만, 리눅스의 진정한 매력은 직접 사용해보면서 느낄 수 있습니다. 키보드로 명령어를 입력하고, 시스템이 어떻게 반응하는지 관찰하며, 원하는 작업을 수행해내는 경험만큼 확실한 학습 방법은 없습니다.

이번 Chapter 2에서는 본격적인 리눅스 탐험을 위한 **실습 환경**을 만드는 다양한 방법들을 알아보고, 여러분의 상황에 맞는 최적의 방법을 선택하여 준비하는 과정을 안내합니다. 마치 요리를 배우기 전에 주방을 준비하고 재료를 다듬는 것처럼, 리눅스 학습을 위한 든든한 베이스캠프를 구축하는 단계라고 생각하시면 됩니다.

(참고: 이 글은 2025년 4월 8일을 기준으로 작성되었으므로, 특정 소프트웨어의 버전이나 클라우드 서비스의 인터페이스는 시간이 지남에 따라 변경될 수 있습니다. 항상 공식 문서를 함께 참고하는 습관을 들이는 것이 좋습니다.)


## 2.1 어떤 방법으로 리눅스를 만날까요? (실습 환경 선택)

리눅스를 경험하는 방법은 여러 가지가 있습니다. 각 방법마다 장단점이 있으므로, 여러분의 컴퓨터 사양, 주로 사용하는 운영체제, 학습 목표 등을 고려하여 가장 적합한 방법을 선택하는 것이 중요합니다.


### 2.1.1 가상 머신(Virtual Machine): 내 컴퓨터 안의 또 다른 컴퓨터

가장 인기 있고 초보자에게 추천하는 방법 중 하나입니다. 가상 머신은 현재 사용 중인 운영체제(예: Windows, macOS) 안에 **소프트웨어적으로 또 다른 컴퓨터를 만드는 기술**입니다. 이 가상의 컴퓨터에 리눅스를 설치하여, 마치 별도의 컴퓨터처럼 사용할 수 있습니다.

* **개념:** 하드웨어를 에뮬레이션하는 소프트웨어(하이퍼바이저)를 이용하여, 현재 OS 환경에 영향을 주지 않고 독립된 공간에 다른 OS(리눅스)를 설치 및 실행합니다.
* **주요 소프트웨어:**
  * `VirtualBox`: Oracle에서 제공하는 무료 오픈소스 가상 머신 소프트웨어입니다. 개인 사용자에게 가장 인기 있습니다.
  * `VMware Workstation Player` (Windows/Linux) / `VMware Fusion` (macOS): VMware사에서 제공하며, Player는 개인 사용자의 비상업적 용도에 한해 무료입니다. Fusion은 유료입니다. 성능과 기능 면에서 우수하다는 평가를 받습니다.
* **장점:**
  * **안전함:** 가상 머신 안에서의 작업은 원래 사용하던 운영체제(호스트 OS)에 영향을 주지 않습니다. 마음껏 실험해볼 수 있습니다.
  * **다양한 배포판 경험:** 여러 종류의 리눅스 배포판을 쉽게 설치하고 비교해볼 수 있습니다.
  * **스냅샷 기능:** 특정 시점의 시스템 상태를 저장(스냅샷)했다가 문제가 생기면 해당 시점으로 쉽게 되돌릴 수 있습니다.
  * **설치 과정 학습:** 리눅스 운영체제의 설치 과정을 직접 경험해볼 수 있습니다.
* **단점:**
  * **자원 요구:** 컴퓨터의 `RAM`, `CPU`, 디스크 공간을 추가로 사용합니다. 사양이 낮은 컴퓨터에서는 다소 느릴 수 있습니다.
  * **성능:** 실제 하드웨어에 직접 설치하는 것보다는 약간의 성능 저하가 있을 수 있습니다.
* **추천 대상:** 데스크탑이나 노트북을 사용하는 대부분의 리눅스 입문자에게 가장 좋은 출발점입니다.


### 2.1.2 클라우드 서비스: 언제 어디서든 리눅스 서버

클라우드 컴퓨팅 서비스를 이용하여 **인터넷 상의 가상 서버(인스턴스)** 에 리눅스를 설치하고 원격으로 접속하여 사용하는 방법입니다.

* **개념:** AWS, Google Cloud, Azure 같은 클라우드 제공 업체로부터 가상 서버 자원을 빌려서 사용합니다. 사용자는 자신의 컴퓨터에서 `SSH`라는 원격 접속 프로토콜을 통해 이 서버에 접속하여 리눅스 명령어를 실행합니다.
* **주요 서비스:**
  * `Amazon Web Services (AWS) EC2`
  * `Google Cloud Platform (GCP) Compute Engine`
  * `Microsoft Azure Virtual Machines`
  * `DigitalOcean`, `Linode`, `Vultr` 등 (더 저렴하고 사용이 간편한 VPS 서비스)
* **장점:**
  * **접근성:** 인터넷만 연결되어 있으면 어디서든 접속 가능합니다.
  * **확장성:** 필요에 따라 서버 사양(CPU, RAM, 디스크)을 쉽게 조절할 수 있습니다.
  * **실제 서버 환경:** 실제 서비스가 운영되는 환경과 매우 유사한 경험을 할 수 있습니다.
  * **무료 체험(Free Tier):** 대부분의 주요 클라우드 제공 업체는 신규 가입자에게 일정 기간 또는 일정 사양까지 무료로 사용할 수 있는 플랜을 제공합니다.
* **단점:**
  * **인터넷 연결 필수:** 오프라인 상태에서는 사용할 수 없습니다.
  * **비용 발생 가능성:** 무료 사용량 한도를 초과하면 비용이 발생할 수 있습니다. (주의 필요!)
  * **설치 과정 생략:** 운영체제 설치 과정을 직접 경험하기는 어렵습니다. (미리 준비된 이미지를 사용)
* **추천 대상:** 실제 서버 환경을 경험하고 싶은 사용자, 웹 애플리케이션 배포 등을 염두에 둔 개발자, 혹은 개인 컴퓨터 사양이 부족한 경우 좋은 대안입니다.


### 2.1.3 WSL(Windows Subsystem for Linux): 윈도우에서 리눅스 사용하기

Windows 10 및 Windows 11 사용자를 위한 아주 편리한 옵션입니다. 별도의 가상 머신 프로그램 설치 없이, **Windows 시스템 안에서 직접 리눅스 환경을 실행**할 수 있게 해주는 기능입니다.

* **개념:** Windows 기능 중 하나로, 리눅스 배포판(주로 명령줄 환경)을 Windows 스토어 등에서 설치하여 바로 사용할 수 있습니다. WSL 1과 WSL 2가 있으며, **WSL 2**는 실제 리눅스 커널을 사용하여 호환성과 성능이 훨씬 뛰어납니다 (특별한 이유가 없다면 WSL 2 사용 권장).
* **장점:**
  * **간편한 설치 및 사용:** 몇 가지 명령어 입력만으로 빠르게 리눅스 환경을 구축할 수 있습니다.
  * **Windows와의 통합:** Windows 파일 시스템에 쉽게 접근할 수 있으며, Windows와 리눅스 간의 연동이 편리합니다.
  * **뛰어난 성능 및 낮은 자원 사용량:** 일반적인 가상 머신보다 가볍고 빠르게 작동합니다.
* **단점:**
  * **Windows 전용:** 당연하지만 Windows 운영체제에서만 사용할 수 있습니다.
  * **GUI 제한적 (과거):** 초기에는 명령줄 위주였으나, 현재 WSL 2에서는 리눅스 GUI 애플리케이션 실행도 지원합니다. 하지만 데스크탑 환경 전체를 매끄럽게 사용하는 경험과는 차이가 있을 수 있습니다.
  * **일부 하드웨어 접근 제한:** 특정 하드웨어(예: GPU 연산)를 직접 제어하는 작업에는 제약이 있을 수 있습니다 (개선 중).
* **추천 대상:** Windows 사용자, 특히 개발 관련 작업을 위해 리눅스 명령줄 도구나 환경이 필요하지만, 완전한 별도 OS 설치는 부담스러운 경우 최고의 선택입니다.


### 2.1.4 듀얼 부팅 / 직접 설치: 전문가를 위한 선택 (주의사항 포함)

컴퓨터의 하드 드라이브에 **리눅스를 직접 설치**하는 방법입니다. 기존 운영체제(예: Windows)를 유지하면서 리눅스를 추가로 설치하여, 부팅 시 어떤 운영체제를 사용할지 선택하는 **듀얼 부팅(Dual Boot)** 구성이 일반적입니다. 또는 컴퓨터 전체를 리눅스 전용으로 사용할 수도 있습니다.

* **개념:** 하드 드라이브의 공간(파티션)을 나누어 리눅스를 설치하고, 컴퓨터가 켜질 때 부트 로더(Boot Loader)를 통해 사용할 운영체제를 선택합니다.
* **장점:**
  * **최상의 성능:** 가상화 오버헤드 없이 하드웨어 성능을 100% 활용할 수 있습니다.
  * **완벽한 하드웨어 접근:** 모든 하드웨어 장치에 직접 접근하고 제어할 수 있습니다.
* **단점:**
  * **설치 복잡성 및 위험:** 파티션 설정 등이 복잡하며, 실수할 경우 기존 데이터가 손실될 위험이 있습니다. **매우 주의가 필요하며, 중요한 데이터는 반드시 백업해야 합니다.**
  * **OS 전환 시 재부팅 필요:** Windows와 리눅스를 번갈아 사용하려면 매번 컴퓨터를 재부팅해야 합니다.
  * **드라이버 호환성 문제:** 일부 최신 하드웨어나 특수 장치의 리눅스 드라이버 지원이 부족할 수 있습니다.
* **추천 대상:** **절대 초보자에게 권장하지 않습니다.** 리눅스를 주 운영체제로 사용하고 싶거나, 하드웨어 성능이 매우 중요하며, 파티션 및 부팅 과정에 대한 이해가 있는 숙련된 사용자에게 적합합니다.

---

**어떤 방법을 선택해야 할까요?**

* **가장 무난하고 안전한 시작:** **가상 머신 (VirtualBox 추천)**
* **Windows 사용자이고 개발 목적이 강하다면:** **WSL 2**
* **서버 환경 경험 및 원격 작업:** **클라우드 서비스 (무료 티어 활용)**
* **(주의!) 리눅스를 깊게 파고들고 싶고 위험을 감수할 수 있다면:** **듀얼 부팅 / 직접 설치**

## 2.2 [선택] VirtualBox/VMware에 리눅스 설치하기 (개요)

가상 머신에 리눅스를 설치하는 과정은 비교적 간단하며, 대부분의 리눅스 배포판이 친절한 그래픽 설치 프로그램을 제공합니다. 전체적인 단계는 다음과 같습니다.

1. **가상 머신 소프트웨어 설치:** VirtualBox 또는 VMware Workstation Player 웹사이트에서 프로그램을 다운로드하여 설치합니다.
2. **리눅스 배포판 ISO 파일 다운로드:** 설치하고 싶은 리눅스 배포판(예: Ubuntu Desktop LTS)의 공식 웹사이트에서 `.iso` 확장자를 가진 설치 이미지를 다운로드합니다.
3. **새 가상 머신 생성:** VirtualBox/VMware를 실행하고 '새로 만들기' 또는 'Create a New Virtual Machine'을 선택합니다.
    * 가상 머신의 이름, 종류(Linux), 버전(예: Ubuntu 64-bit)을 지정합니다.
    * `RAM`(메모리), `CPU` 코어 수, 가상 하드 디스크 크기를 할당합니다. (배포판 권장 사양 이상으로 설정, 예: RAM 4GB 이상, 디스크 25GB 이상)
4. **ISO 파일 연결:** 생성된 가상 머신 설정에서 '저장소' 또는 'CD/DVD' 항목을 찾아, 다운로드한 리눅스 `.iso` 파일을 가상 광학 드라이브에 연결합니다.
5. **가상 머신 시작 및 리눅스 설치:** 가상 머신을 시작하면 `.iso` 파일로 부팅되어 리눅스 설치 프로그램이 실행됩니다. 화면의 안내에 따라 언어, 키보드 레이아웃, 시간대 등을 선택합니다.
    * **파티션 설정:** 초보자는 **'디스크 전체 사용' 또는 '자동 파티션 설정'** 옵션을 선택하는 것이 안전합니다. (가상 디스크이므로 걱정 없이 진행해도 됩니다.)
    * **사용자 계정 생성:** 리눅스에서 사용할 사용자 이름과 비밀번호를 설정합니다. 이 계정은 `sudo` 권한을 갖는 관리자 계정이 됩니다.
6. **설치 완료 및 재부팅:** 설치가 완료되면 가상 머신을 재부팅하라는 메시지가 나옵니다. 재부팅 전에 연결했던 `.iso` 파일을 가상 드라이브에서 제거해야 합니다.
7. **(권장) 게스트 확장/VMware Tools 설치:** 리눅스 설치 후, 가상 머신 메뉴에서 '게스트 확장 설치' 또는 'Install VMware Tools'를 실행합니다. 이는 호스트 OS와 클립보드 공유, 화면 크기 자동 조절, 파일 공유 등의 편의 기능을 제공합니다.

> **팁:** 특정 배포판(예: Ubuntu)과 가상 머신 소프트웨어(예: VirtualBox) 조합으로 설치 과정을 검색하면(예: "VirtualBox Ubuntu 22.04 설치 가이드"), 스크린샷과 함께 자세히 설명된 블로그 글이나 유튜브 영상을 많이 찾을 수 있습니다.


## 2.3 [선택] 클라우드에 리눅스 인스턴스 생성하기 (개요)

클라우드 서비스를 이용하는 경우, 웹 기반의 관리 콘솔을 통해 몇 번의 클릭만으로 리눅스 서버를 생성(프로비저닝)할 수 있습니다. AWS EC2를 예시로 한 일반적인 단계는 다음과 같습니다.

1. **클라우드 서비스 가입:** AWS, GCP, Azure 등의 웹사이트에서 계정을 생성합니다. (무료 티어 가입 시 해외 결제가 가능한 신용카드 정보가 필요할 수 있습니다. 실제 과금은 무료 한도 초과 시에만 발생합니다.)
2. **관리 콘솔 접속:** 가입한 계정으로 로그인하여 관리 콘솔(Management Console)에 접속합니다.
3. **가상 서버 서비스 선택:** AWS의 경우 `EC2`, GCP는 `Compute Engine`, Azure는 `Virtual Machines` 메뉴로 이동합니다.
4. **인스턴스 생성/시작:** '인스턴스 시작' 또는 '만들기' 버튼을 클릭합니다.
5. **OS 이미지(AMI) 선택:** 사용할 리눅스 배포판 이미지를 선택합니다. (예: Ubuntu Server LTS, Amazon Linux 등 - 무료 티어 사용 가능 이미지를 확인하세요.)
6. **인스턴스 유형 선택:** 서버의 사양(vCPU, 메모리)을 선택합니다. 무료 티어에서 제공하는 유형(예: AWS `t2.micro`)을 선택합니다.
7. **네트워크 및 스토리지 설정:** 일반적으로 기본 설정을 사용해도 무방하지만, 필요에 따라 가상 네트워크(VPC), 서브넷, 디스크 크기 등을 조정할 수 있습니다.
8. **보안 그룹(방화벽) 설정:** **매우 중요!** 인스턴스에 접근할 수 있는 네트워크 포트를 설정합니다. 최소한 `SSH`(포트 22) 접근을 허용해야 하며, 보안을 위해 '내 IP'에서만 접근 가능하도록 설정하는 것이 좋습니다.
9. **SSH 키 페어 생성/선택:** **매우 중요!** 인스턴스에 접속할 때 사용할 암호화 키 파일을 생성하거나 기존 키를 선택합니다. 새 키를 생성했다면 **`.pem` 확장자의 개인 키 파일을 즉시 다운로드하여 안전한 곳에 보관**해야 합니다. 이 파일이 없으면 서버에 접속할 수 없습니다!
10. **인스턴스 시작:** 설정을 검토하고 '시작' 버튼을 누르면 몇 분 안에 리눅스 서버 인스턴스가 생성되고 실행됩니다.
11. **SSH 접속:** 생성된 인스턴스의 **공개 IP 주소(Public IP Address)** 와 다운로드한 **`.pem` 키 파일**을 이용하여 SSH 클라이언트로 접속합니다.
    * Linux/macOS/WSL 터미널: `ssh -i /경로/내키파일.pem 사용자명@인스턴스_IP주소` (예: `ssh -i mykey.pem ubuntu@13.125.100.1`)
    * Windows: `PuTTY` 같은 SSH 클라이언트 프로그램을 사용하여 IP 주소와 키 파일을 설정하여 접속합니다. (PuTTY는 `.pem` 파일을 자체 형식 `.ppk`로 변환해야 합니다.)

> **팁:** 각 클라우드 제공 업체의 공식 문서나 시작 가이드에 가장 정확하고 최신 정보가 담겨 있습니다. 'AWS EC2 Ubuntu 인스턴스 생성 가이드' 등으로 검색하면 많은 자료를 찾을 수 있습니다.


## 2.4 [선택] WSL 설치 및 사용하기 (개요)

최신 Windows 10 또는 11 사용자라면 WSL을 설치하는 것이 가장 간편할 수 있습니다.

1. **관리자 권한으로 터미널 실행:** 시작 메뉴에서 'PowerShell' 또는 '명령 프롬프트'를 찾아 마우스 오른쪽 버튼을 클릭하고 '관리자 권한으로 실행'을 선택합니다.
2. **WSL 설치 명령어 실행:** 열린 터미널 창에 다음 명령어를 입력하고 Enter 키를 누릅니다.

    ```powershell
    wsl --install
    ```

    * 이 명령어는 필요한 Windows 기능을 활성화하고, 최신 리눅스 커널을 다운로드하며, WSL 2를 기본 버전으로 설정하고, 기본 배포판(주로 Ubuntu)을 설치합니다.
3. **(선택) 특정 배포판 설치:** 만약 Ubuntu가 아닌 다른 배포판(예: Debian)을 설치하고 싶다면, 설치 가능한 배포판 목록을 확인(`wsl --list --online`)한 후 다음 형식으로 명령어를 실행합니다.

    ```powershell
    wsl --install -d <배포판이름>
    ```

    (예: `wsl --install -d Debian`)
4. **시스템 재부팅 (필요시):** 설치 과정에서 컴퓨터를 재부팅하라는 메시지가 나타날 수 있습니다. 메시지에 따라 재부팅합니다.
5. **리눅스 사용자 설정:** 재부팅 후 또는 설치 완료 후, 설치된 리눅스 배포판이 자동으로 실행되어 초기 설정을 진행합니다. 리눅스 환경에서 사용할 **사용자 이름(username)** 과 **비밀번호(password)** 를 입력하라는 메시지가 나타나면 원하는 대로 설정합니다. (Windows 계정과 달라도 됩니다.)
6. **WSL 실행:** 설정이 완료되면 언제든지 시작 메뉴에서 설치된 리눅스 배포판 이름(예: 'Ubuntu')을 검색하여 실행하거나, Windows 터미널 앱에서 새 탭으로 열 수 있습니다.

> **팁:** Microsoft의 공식 WSL 문서([https://learn.microsoft.com/ko-kr/windows/wsl/install]([https://learn.microsoft.com/ko-kr/windows/wsl/install))에]([https://www.google.com/search?q=https://learn.microsoft.com/ko-kr/windows/wsl/install))%EC%97%90](https://www.google.com/search?q=https://learn.microsoft.com/ko-kr/windows/wsl/install))%EC%97%90)) 가장 정확하고 상세한 설치 및 사용법이 안내되어 있습니다.


## 2.5 설치 후 초기 설정 (Post-Installation Steps)

어떤 방법으로든 리눅스 환경을 성공적으로 준비했다면, 본격적인 사용에 앞서 몇 가지 기본적인 초기 설정을 해주는 것이 좋습니다. 이제 여러분의 리눅스 터미널에서 다음 명령어들을 실행해 보세요.

1. **시스템 업데이트:** 가장 먼저 해야 할 일은 설치된 소프트웨어 패키지 목록을 최신화하고, 기존 패키지들을 최신 버전으로 업데이트하는 것입니다. 보안 및 안정성을 위해 필수적입니다.
    * Debian/Ubuntu 계열:

        ```bash
        sudo apt update && sudo apt upgrade -y
        ```

    * Fedora/CentOS/Rocky/Alma 계열 (dnf 사용 시):

        ```bash
        sudo dnf update -y
        ```

    * CentOS 7 등 (yum 사용 시):

        ```bash
        sudo yum update -y
        ```

    * `sudo`: 관리자 권한으로 명령어를 실행하겠다는 의미입니다. 비밀번호를 물어보면 리눅스 사용자 계정의 비밀번호를 입력합니다. (화면에 입력 내용이 표시되지 않는 것이 정상이니 당황하지 마세요!)
    * `apt update` / `dnf update` / `yum update`: 패키지 목록 정보를 새로고침합니다.
    * `apt upgrade` / `dnf update` / `yum update`: 실제 패키지를 최신 버전으로 업그레이드합니다. (`-y` 옵션은 중간에 묻는 질문에 모두 'yes'로 자동 응답하는 옵션입니다.)

2. **네트워크 연결 확인:** 인터넷 연결이 잘 되는지 확인합니다.
    * IP 주소 확인: `ip addr` 또는 `ip address show`
    * 외부 사이트 연결 테스트: `ping google.com` (Ctrl+C를 눌러 중지)

3. **사용자 계정 및 `sudo` 확인:** 설치 시 생성한 사용자 계정으로 로그인되어 있는지, `sudo` 명령어가 잘 작동하는지 확인합니다. (시스템 업데이트 명령어가 성공했다면 `sudo`는 잘 작동하는 것입니다.)

4. **(선택) 필수 도구 설치:** 앞으로의 실습을 위해 몇 가지 기본적인 개발 도구를 미리 설치해두면 편리합니다.
    * Debian/Ubuntu 계열:

        ```bash
        sudo apt install -y build-essential git vim curl wget unzip
        ```

    * Fedora/CentOS/Rocky/Alma 계열:

        ```bash
        sudo dnf groupinstall -y "Development Tools"
        sudo dnf install -y git vim curl wget unzip
        ```

    * `build-essential` 또는 `"Development Tools"`: C/C++ 코드 컴파일 등에 필요한 기본적인 도구 모음입니다.
    * `git`: 버전 관리 시스템입니다.
    * `vim`: 강력한 텍스트 편집기입니다. (이미 설치되어 있을 수도 있습니다.)
    * `curl`, `wget`: 웹에서 파일을 다운로드하는 도구입니다.
    * `unzip`: `.zip` 압축 파일을 해제하는 도구입니다.

이제 여러분은 리눅스 명령어를 입력하고 실행 결과를 확인할 준비가 되었습니다! 환경 설정이 처음에는 다소 복잡하게 느껴질 수 있지만, 이 과정을 통해 리눅스와 한 걸음 더 가까워졌다고 생각하시면 좋겠습니다. 만약 특정 단계에서 어려움을 겪는다면, 주저하지 말고 해당 소프트웨어의 공식 문서나 온라인 커뮤니티의 도움을 받아보세요.
