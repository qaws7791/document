지난 Chapter 2에서 우리는 리눅스를 직접 만져볼 수 있는 실습 환경을 성공적으로 구축했습니다. 이제 드디어 리눅스와 본격적으로 대화를 나눌 시간입니다! 앞으로 우리는 주로 키보드로 명령어를 입력하고 텍스트로 결과를 확인하는 방식으로 리눅스와 소통하게 될 것입니다. 특히 서버나 클라우드 환경에서는 이런 **명령줄 인터페이스(Command-Line Interface, CLI)** 가 거의 유일한 소통 창구인 경우가 많습니다.

처음에는 검은 화면에 글자만 깜빡이는 모습이 다소 낯설고 어렵게 느껴질 수도 있습니다. 하지만 걱정 마세요! 몇 가지 기본적인 개념만 이해하고 나면, 명령줄 환경이 얼마나 강력하고 효율적인 작업 방식인지 깨닫게 될 것입니다. 마치 새로운 언어를 배우는 것처럼, 차근차근 익숙해지면 됩니다.

이번 Chapter 3에서는 리눅스와 대화하기 위한 필수 도구인 **터미널(Terminal)** 과 **쉘(Shell)** 이 무엇인지 알아보고, 쉘이 우리에게 보내는 신호인 **프롬프트(Prompt)** 를 이해하며, 아주 기본적인 명령어들을 직접 입력해 보겠습니다. 또한, 앞으로 수많은 명령어들을 배울 때 길잡이가 되어 줄 명령어의 기본 구조와 도움말을 확인하는 방법까지 확실하게 익혀보겠습니다.


## 3.1 GUI vs CLI: 리눅스와 친해지는 방법

컴퓨터와 소통하는 방식은 크게 두 가지로 나눌 수 있습니다.

* **GUI (Graphical User Interface, 그래픽 사용자 인터페이스):** 우리가 일반적으로 Windows나 macOS에서 경험하는 방식입니다. 바탕화면의 아이콘을 마우스로 클릭하고, 메뉴를 통해 프로그램을 실행하며, 그래픽 창 안에서 작업을 수행합니다.
  * **장점:** 시각적이고 직관적이어서 초보자가 사용법을 배우기 쉽습니다.
  * **단점:** 때로는 복잡하거나 반복적인 작업을 수행하기 번거로울 수 있습니다. 그래픽 처리를 위해 더 많은 시스템 자원(CPU, 메모리)을 사용하며, 서버 환경에서는 GUI가 제공되지 않는 경우가 많습니다. (물론, Ubuntu Desktop처럼 뛰어난 GUI를 제공하는 리눅스 배포판도 많습니다!)
* **CLI (Command-Line Interface, 명령줄 인터페이스):** 키보드로 특정 명령어를 텍스트 형태로 입력하면, 시스템이 해당 명령을 수행하고 결과를 텍스트로 보여주는 방식입니다.
  * **장점:** 익숙해지면 매우 빠르고 효율적으로 작업을 처리할 수 있습니다. 여러 명령어를 조합하거나 스크립트로 만들어 복잡한 작업을 자동화하기 용이합니다. 시스템 자원을 적게 사용하며, 거의 모든 리눅스 시스템(특히 서버)에서 기본적으로 제공됩니다. 원하는 작업을 아주 정밀하게 제어할 수 있습니다.
  * **단점:** 처음에는 배워야 할 명령어가 많고, 결과를 즉각적으로 시각화하기 어렵다는 점에서 진입 장벽이 느껴질 수 있습니다.

우리는 앞으로 **CLI 환경**을 중심으로 리눅스를 배우게 됩니다. 왜냐하면 서버 관리, 클라우드 운영, 개발 자동화 등 전문적인 리눅스 활용 분야에서는 CLI가 핵심적인 역할을 하기 때문입니다. 처음에는 어색하더라도 꾸준히 연습하면 금방 익숙해지고 그 강력함에 매료될 것입니다.


## 3.2 터미널(Terminal)이란? (리눅스와 대화하는 창)

그렇다면 CLI 환경은 어디서 만날 수 있을까요? 바로 **터미널(Terminal)** 또는 **터미널 에뮬레이터(Terminal Emulator)** 라는 프로그램을 통해서입니다. 터미널은 우리가 키보드로 명령어를 입력하고, 시스템이 출력하는 텍스트 결과를 눈으로 확인할 수 있도록 **문자 기반의 입출력 환경을 제공하는 응용 프로그램 창**입니다.

마치 리눅스 시스템의 내부와 연결된 전용 타자기 또는 통신 창구라고 생각할 수 있습니다. 우리가 터미널 창에 명령어를 타이핑하면, 그 내용이 리눅스 시스템 내부의 '누군가'에게 전달되는 것이죠.

* **어떻게 열 수 있나요?**
  * Ubuntu Desktop 같은 GUI 환경에서는 프로그램 메뉴에서 'Terminal' 또는 '콘솔' 등을 검색하여 실행할 수 있습니다.
  * WSL 환경에서는 Windows 시작 메뉴에서 설치한 리눅스 배포판 이름(예: `Ubuntu`)을 검색하거나, Windows 터미널 앱을 실행하여 사용할 수 있습니다.
  * 클라우드 서버나 다른 리눅스 PC에 원격으로 접속할 때는 `SSH` 클라이언트 프로그램을 사용하는데, 이 `SSH` 접속 자체가 터미널 환경을 통해 이루어집니다.

터미널을 처음 열면 보통 검은색(또는 다른 배경색) 바탕에 커서(cursor)가 깜빡이고 있고, 그 앞에 몇 가지 문자열이 표시되어 있을 것입니다. 이것이 바로 여러분의 명령을 기다리는 리눅스의 모습입니다.


## 3.3 쉘(Shell)이란? (명령어를 해석하는 비서)

터미널 창 자체는 단순히 글자를 입력하고 보여주는 역할만 합니다. 우리가 터미널에 입력한 명령어를 **실제로 이해하고 해석해서, 운영체제(리눅스 커널)에게 전달하여 실행시키는 프로그램**이 필요한데, 이것이 바로 **쉘(Shell)** 입니다.

* **역할:** 사용자와 리눅스 커널 사이의 인터페이스 역할을 합니다. 사용자의 명령어를 해석하고, 커널에게 해당 작업을 요청하며, 커널이 처리한 결과를 다시 사용자에게 보여줍니다.
* **비유:** 터미널이 리눅스와 연결된 '전화기'라면, 쉘은 전화기 너머에서 우리의 말을 알아듣고 실제 업무를 처리해주는 '비서' 또는 '통역사'와 같습니다. 운영체제 커널의 명령 해석기라고 할 수 있죠.


### 3.3.1 다양한 쉘 종류

리눅스에는 여러 종류의 쉘이 있습니다. 각 쉘마다 약간의 기능 차이나 사용법의 차이가 있을 수 있지만, 기본적인 명령어들은 대부분 호환됩니다.

* **`Bash (Bourne Again SHell)`:** 가장 널리 사용되는 쉘입니다. 대부분의 리눅스 배포판과 macOS에서 기본 쉘로 채택되어 있습니다. 기능이 풍부하고 안정적이어서 이 책에서도 주로 `Bash`를 기준으로 설명합니다.
* **`Zsh (Z Shell)`:** `Bash`를 기반으로 더욱 향상된 기능을 제공하는 쉘입니다. 강력한 자동 완성 기능, 편리한 테마 설정 등으로 인기가 높으며, "Oh My Zsh" 같은 프레임워크와 함께 사용되기도 합니다.
* **`Fish (Friendly Interactive SHell)`:** 이름처럼 사용자 친화적인 기능(자동 제안, 구문 강조 등)을 기본적으로 제공하여 초보자가 사용하기 편리한 쉘입니다.
* **`sh (Bourne Shell)`:** 유닉스의 고전적인 쉘입니다. `Bash`는 이 `sh`를 확장한 것입니다. 오래된 시스템과의 호환성이나 간단한 시스템 스크립트 작성에 가끔 사용됩니다.
* **그 외:** `csh`, `tcsh`, `ksh` 등 역사적으로 사용되었거나 특정 목적을 가진 쉘들도 있습니다.


### 3.3.2 우리가 주로 사용할 Bash 쉘

이 책의 모든 예제와 설명은 특별한 언급이 없는 한 **`Bash` 쉘**을 기준으로 합니다. 여러분이 어떤 리눅스 배포판을 설치했든, 거의 대부분 기본 쉘이 `Bash`로 설정되어 있을 가능성이 높습니다.

현재 사용 중인 쉘이 무엇인지 궁금하다면, 터미널에 다음 명령어를 입력해보세요.

```bash
echo $SHELL
```

아마 `/bin/bash` 와 유사한 경로가 출력될 것입니다.


## 3.4 첫 명령어 입력해보기 (프롬프트 이해, 기본 명령어)

이제 터미널을 열고 쉘과 직접 대화해 봅시다. 터미널을 열면 커서 앞에 몇 가지 글자가 보일 텐데, 이것을 **프롬프트(Prompt)** 라고 부릅니다. 프롬프트는 쉘이 여러분의 명령을 입력받을 준비가 되었음을 알리는 신호입니다.

* **프롬프트의 모습:** 프롬프트는 시스템 설정에 따라 다르게 보일 수 있지만, 보통 다음과 같은 정보를 포함합니다.
  * `username@hostname:~/current_directory$`
  * `username`: 현재 로그인한 사용자 이름
  * `hostname`: 컴퓨터의 이름 (네트워크 상에서 식별되는 이름)
  * `current_directory`: 현재 작업 중인 디렉토리(폴더)의 위치 (`~` 표시는 보통 홈 디렉토리를 의미합니다)
  * `$`: 현재 사용자가 일반 사용자임을 나타냅니다.
  * 만약 프롬프트 마지막 문자가 `$` 대신 `#` 이라면, 현재 **루트(root)** 라는 슈퍼 관리자 권한으로 접속된 상태임을 의미합니다. 루트 사용자는 시스템의 모든 권한을 가지므로, **`#` 프롬프트 상태에서는 명령어를 입력할 때 매우 신중해야 합니다!**

* **명령어 입력과 실행:** 프롬프트 뒤에 원하는 명령어를 타이핑하고 `Enter` 키를 누르면 쉘이 해당 명령어를 실행합니다. 명령어 실행이 끝나면 쉘은 결과(출력)를 화면에 보여주고, 다시 다음 명령을 기다리는 새로운 프롬프트를 표시합니다.

자, 이제 몇 가지 아주 간단하고 안전한 명령어를 직접 입력해 봅시다.

1. **현재 날짜와 시간 확인:** `date`

    ```bash
    date
    ```

    (Enter 키를 누르면 현재 시스템의 날짜와 시간이 출력됩니다.)

2. **이번 달 달력 보기:** `cal`

    ```bash
    cal
    ```

    (Enter 키를 누르면 현재 달의 달력이 예쁘게 출력됩니다.)

3. **현재 작업 디렉토리(폴더) 확인:** `pwd` (Print Working Directory)

    ```bash
    pwd
    ```

    (Enter 키를 누르면 현재 내가 위치한 디렉토리의 전체 경로가 출력됩니다. 앞으로 파일을 다룰 때 매우 중요한 명령어입니다.)

4. **터미널 화면 깨끗하게 지우기:** `clear`

    ```bash
    clear
    ```

    (Enter 키를 누르면 터미널 화면의 내용이 지워지고 프롬프트만 맨 위에 표시됩니다. 이전 내용이 사라지는 것은 아니고 화면만 정리하는 것입니다.)

어떤가요? 직접 명령어를 입력하고 결과를 확인하니 조금은 리눅스와 친해진 느낌이 드시나요?


## 3.5 명령어 구조 이해하기 (명령어, 옵션, 인자)

리눅스 명령어는 보통 다음과 같은 기본 구조를 가집니다. 물론 명령어에 따라 옵션이나 인자가 없을 수도 있습니다.

`command [options] [arguments]`

* **`command` (명령어):** 실행하고자 하는 작업의 이름입니다. (예: `ls`, `pwd`, `cp`, `mkdir`) 보통 명령어 줄의 가장 앞에 위치합니다.
* **`options` (옵션):** 명령어의 기본적인 동작 방식을 변경하거나 특정 기능을 추가/제어하는 역할을 합니다.
  * 보통 하이픈(`-`) 하나 뒤에 영문자 하나가 오거나(예: `-l`, `-a`),
  * 하이픈 두 개(`--`) 뒤에 단어 형태(예: `--all`, `--human-readable`)로 사용됩니다.
  * 하이픈 하나짜리 옵션은 여러 개를 붙여서 사용할 수도 있습니다. (예: `ls -l -a` 는 `ls -la` 와 동일하게 작동합니다.)
  * 옵션은 명령어의 동작을 세밀하게 조정할 때 사용됩니다. (예: `ls` 명령어에 `-l` 옵션을 주면 파일 목록을 자세한 정보와 함께 보여줍니다.)
* **`arguments` (인자 또는 인수):** 명령어가 작업을 수행할 대상(예: 파일 이름, 디렉토리 경로 등)을 지정합니다.
  * 옵션 뒤에 오거나, 옵션이 없다면 명령어 바로 뒤에 옵니다.
  * 명령어에 따라 여러 개의 인자를 가질 수도 있습니다. (예: `cp source.txt destination.txt` 에서 `source.txt`와 `destination.txt`는 `cp` 명령어의 인자입니다.)

**예시:**

* `ls`: 명령어만 사용 (현재 디렉토리의 파일 목록을 기본 형태로 보여줌)
* `ls -l`: 명령어 + 옵션 (현재 디렉토리의 파일 목록을 긴 형식(long format)으로 보여줌)
* `ls /etc`: 명령어 + 인자 (`/etc` 디렉토리의 파일 목록을 보여줌)
* `ls -a /home`: 명령어 + 옵션 + 인자 (`/home` 디렉토리의 숨김 파일 포함 모든 파일 목록을 보여줌)

**중요:** 명령어, 옵션, 인자 사이는 **띄어쓰기(공백)** 로 구분합니다!


## 3.6 도움말 활용하기 (`man`, `info`, `--help` 옵션)

리눅스에는 수천 개의 명령어가 있고, 각 명령어마다 수많은 옵션이 존재합니다. 이 모든 것을 외우는 것은 불가능하고, 그럴 필요도 없습니다. 중요한 것은 **필요할 때 원하는 정보를 스스로 찾아보는 능력**입니다. 다행히 리눅스는 강력한 도움말 시스템을 내장하고 있습니다.

1. **`--help` 옵션:** 대부분의 명령어는 `--help` 라는 옵션을 제공합니다. 명령어 뒤에 이 옵션을 붙여 실행하면, 해당 명령어의 간단한 사용법과 주요 옵션 목록을 터미널에 바로 출력해줍니다. 빠르게 사용법을 확인하고 싶을 때 유용합니다. (가끔 `-h` 옵션으로 제공되기도 합니다.)

    ```bash
    ls --help
    mkdir --help
    cp --help
    ```

2. **`man` 명령어 (Manual):** 리눅스의 공식 사용 설명서인 **매뉴얼 페이지(Manual page)** 를 보여주는 명령어입니다. 명령어의 이름, 사용 형식, 상세 설명, 모든 옵션, 예제 등 매우 자세한 정보를 제공합니다.

    ```bash
    man ls
    man cp
    man passwd
    ```

    * `man` 페이지는 내용이 길기 때문에 한 화면에 다 보이지 않습니다.
        * **스크롤:** 방향키(↑, ↓), Page Up, Page Down 키 사용
        * **검색:** `/` 키를 누르고 검색할 단어 입력 후 Enter (n: 다음 찾기, N: 이전 찾기)
        * **종료:** `q` 키를 누르면 `man` 페이지 보기를 종료하고 원래 터미널로 돌아갑니다.
    * `man` 페이지는 섹션(section)으로 구분되어 있습니다. (예: 1=사용자 명령어, 5=설정 파일 형식 등). 가끔 이름이 같은 다른 종류의 정보가 있을 때 `man 1 ls` 또는 `man 5 passwd` 처럼 섹션 번호를 명시하여 원하는 매뉴얼을 볼 수 있습니다.

3. **`info` 명령어:** `man` 보다 더 구조화되고 하이퍼링크 개념이 도입된 형태의 문서를 보여줍니다. 일부 명령어는 `man` 페이지보다 `info` 문서가 더 자세한 경우도 있습니다. 하지만 처음 사용하기에는 `man` 보다 조금 복잡하게 느껴질 수 있습니다.

    ```bash
    info ls
    info coreutils
    ```

    (이동 및 종료 방법은 `man`과 유사하며, 내부적으로 하이퍼링크 이동 기능 등이 있습니다.)

4. **(참고) `whatis`, `apropos` 명령어:** 특정 명령어의 아주 간단한 설명을 보여주거나(`whatis ls`), 특정 키워드와 관련된 명령어 목록을 찾아볼 때(`apropos directory`) 사용합니다.

**가장 중요한 도움말 도구:** `--help` 와 `man` 입니다. 앞으로 새로운 명령어를 배울 때마다 이 도움말 기능을 적극적으로 활용하는 습관을 들이세요!

이제 여러분은 리눅스와 소통하기 위한 기본적인 준비를 마쳤습니다. 터미널 창을 열고, 쉘 프롬프트를 이해하고, 간단한 명령어를 입력하고 결과를 확인했으며, 명령어의 구조와 도움말을 찾는 방법까지 배웠습니다. 비록 아직은 몇 가지 명령어만 사용해봤지만, 이것이 바로 리눅스 세계를 탐험하는 첫걸음입니다.
