지난 Chapter 4에서는 리눅스 파일 시스템을 탐험하고 파일을 관리하는 기본적인 명령어들을 익혔습니다. 파일을 생성하고(`touch`), 내용을 확인하고(`cat`, `less`), 복사하거나(`cp`) 이동(`mv`)하는 방법을 배웠죠. 하지만 파일의 내용을 직접 **수정하거나 새로운 내용을 작성**하려면 어떻게 해야 할까요? 바로 **텍스트 편집기(Text Editor)** 가 필요한 순간입니다.

물론 Ubuntu Desktop 같은 그래픽 환경에서는 Gedit, VS Code 등 다양한 그래픽 텍스트 편집기를 사용할 수 있습니다. 하지만 리눅스를 깊이 사용하다 보면, 특히 서버 환경이나 원격 접속 상황에서는 그래픽 환경 없이 **터미널 안에서 직접 텍스트 파일을 편집**해야 하는 경우가 매우 많습니다. 설정 파일을 수정하거나, 쉘 스크립트를 작성하거나, 간단한 메모를 남길 때 터미널 기반의 텍스트 편집기는 필수적인 도구입니다.

이번 Chapter 5에서는 리눅스 환경에서 가장 널리 사용되는 두 가지 터미널 기반 텍스트 편집기, **Vim(빔)** 과 **Nano(나노)** 의 기본적인 사용법을 알아보겠습니다.


## 5.1 왜 터미널에서 텍스트 편집을 할까요? (Why Edit Text in the Terminal?)

이미 편리한 그래픽 편집기들이 많은데, 왜 굳이 터미널에서 텍스트 편집하는 방법을 배워야 할까요? 몇 가지 중요한 이유가 있습니다.

1. **서버 환경 접근성:** 대부분의 리눅스 서버에는 자원을 아끼기 위해 그래픽 환경(GUI)이 설치되어 있지 않습니다. 서버에 `SSH`로 원격 접속하면 오직 터미널(CLI) 환경만 주어지는데, 이때 설정 파일 등을 수정하려면 터미널 편집기를 사용할 수밖에 없습니다.
2. **작업 효율성 (WSL/개발):** 개발자들은 종종 터미널에서 컴파일, 빌드, 버전 관리 등 다양한 작업을 수행합니다. 이때 간단한 코드 수정이나 설정 변경을 위해 굳이 그래픽 편집기로 전환하지 않고 터미널 내에서 바로 편집기를 실행하면 작업 흐름이 끊기지 않고 효율적입니다.
3. **자원 효율성:** 터미널 편집기는 그래픽 편집기에 비해 시스템 자원(CPU, 메모리)을 훨씬 적게 사용합니다. 저사양 시스템이나 원격 환경에서 유리합니다.
4. **보편성:** `Vim`(또는 그 이전 버전인 `vi`)과 `Nano`는 거의 모든 리눅스 및 유닉스 계열 시스템에 기본적으로 설치되어 있거나 쉽게 설치할 수 있습니다. 하나만 제대로 배워두면 어떤 환경에 가더라도 텍스트 파일을 편집할 수 있습니다.

따라서 리눅스를 제대로 활용하고자 한다면, 최소한 하나의 터미널 기반 텍스트 편집기 사용법을 익혀두는 것이 매우 중요합니다.


## 5.2 강력한 편집기 Vim (초보자를 위한 친절한 안내) (Powerful Editor Vim: A Gentle Guide for Beginners)

**Vim(빔)** 은 오랜 역사(그 전신인 `vi` 편집기부터)를 가진 매우 강력하고 효율적인 텍스트 편집기입니다. 수많은 리눅스/유닉스 시스템에 기본적으로 포함되어 있으며, 개발자들 사이에서 특히 인기가 높습니다. 키보드만으로 거의 모든 편집 작업을 수행할 수 있어 익숙해지면 매우 빠른 작업 속도를 자랑합니다.

**하지만 솔직히 말해서, Vim은 초보자에게는 배우기 어려운 편집기로 악명(?)이 높습니다.** 일반적인 편집기와는 다른 독특한 '모드(Mode)' 개념을 가지고 있기 때문입니다. 처음에는 좌절감을 느낄 수도 있지만, 약간의 인내심을 가지고 기본적인 사용법만 익혀두면 그 강력함에 매료될 것이고, 어떤 리눅스 환경에서도 당황하지 않고 파일을 편집할 수 있는 든든한 무기가 될 것입니다.

* **Vim 시작하기:**
  * `vim <파일 이름>`: 기존 파일을 열거나, 파일이 없으면 새 파일을 만들 준비를 합니다.
  * `vim`: 파일 이름 없이 Vim만 실행합니다.


### 5.2.1 Vim의 모드 이해하기 (Understanding Vim Modes)

Vim을 이해하는 가장 중요한 열쇠는 바로 **'모드(Mode)'** 개념입니다. Vim은 현재 어떤 모드에 있는지에 따라 키 입력의 의미가 달라집니다.

1. **일반 모드 (Normal Mode):**
    * Vim을 처음 시작했을 때의 기본 모드입니다.
    * 이 모드에서는 **키보드 입력이 바로 텍스트로 입력되지 않습니다.** 대신, 커서 이동, 텍스트 삭제(`d`), 복사(`y`), 붙여넣기(`p`), 다른 모드로 전환 등 **명령(Command)** 을 내리는 데 사용됩니다.
    * 다른 모드에서 작업하다가 `Esc` 키를 누르면 대부분 일반 모드로 돌아옵니다. **Vim 사용 중에는 `Esc` 키를 자주 누르는 습관을 들이는 것이 좋습니다.** (일반 모드가 Vim의 '홈 베이스'입니다.)

2. **입력 모드 (Insert Mode):**
    * 실제 파일에 **텍스트를 입력**할 수 있는 모드입니다. 우리가 일반적으로 생각하는 텍스트 편집기 상태와 유사합니다.
    * 일반 모드에서 `i`, `a`, `o` 등의 키를 눌러 입력 모드로 전환합니다. (아래 설명 참조)
    * 입력 모드일 때는 보통 화면 하단에 `-- INSERT --` 와 같은 표시가 나타납니다.
    * 입력 모드에서 일반 모드로 돌아가려면 `Esc` 키를 누릅니다.

3. **명령-라인 모드 (Command-Line Mode):**
    * **파일 저장(`:w`), 종료(`:q`), 검색(`/`, `?`), 치환(`:s`)** 등 Vim의 확장 기능을 실행할 때 사용합니다.
    * 일반 모드에서 콜론(`:`) 키를 누르면 화면 하단에 `:` 가 나타나면서 이 모드로 진입합니다. 여기에 명령어를 입력하고 Enter 키를 누릅니다.
    * 명령 실행 후 보통 자동으로 일반 모드로 돌아갑니다.

4. **(참고) 비주얼 모드 (Visual Mode):**
    * 텍스트 블록을 시각적으로 선택할 때 사용합니다. 일반 모드에서 `v`(문자 단위), `V`(줄 단위), `Ctrl+v`(블록 단위) 키로 진입합니다. 선택 후 `d`(삭제), `y`(복사) 등의 작업을 할 수 있습니다. `Esc` 키로 일반 모드로 돌아갑니다. (초보 단계에서는 필수는 아닙니다.)

**핵심:** 내가 지금 **어떤 모드에 있는지 항상 인지**하는 것이 중요합니다. 뭔가 잘못 입력되는 것 같으면 일단 `Esc` 키를 눌러 **일반 모드**로 돌아온 다음 다시 시도하세요.


### 5.2.2 기본적인 이동, 입력, 저장, 종료 방법 (Basic Navigation, Input, Saving, Quitting)

Vim에서 살아남기 위한 최소한의 필수 기술입니다. **아래 명령어들은 대부분 일반 모드(Normal Mode)에서 실행합니다.**

* **커서 이동 (일반 모드):**
  * **방향키:** 대부분 시스템에서 작동하지만, Vim 고수들은 사용하지 않습니다. 😉
  * **`h`**: 왼쪽으로 이동
  * **`j`**: 아래쪽으로 이동
  * **`k`**: 위쪽으로 이동
  * **`l`**: 오른쪽으로 이동 (키보드 오른쪽 자판에 모여 있어 손 이동 없이 빠름)
  * `w`: 다음 단어(word)의 시작으로 이동
  * `b`: 이전 단어(back)의 시작으로 이동
  * `0` (숫자 0): 현재 줄의 맨 처음으로 이동
  * `$`: 현재 줄의 맨 끝으로 이동
  * `gg`: 파일의 맨 처음(Go)으로 이동
  * `G`: 파일의 맨 끝(Go)으로 이동
  * `<숫자>G`: 지정한 `<숫자>`번째 줄로 이동 (예: `10G` -> 10번째 줄로 이동)

* **입력 모드 진입 (일반 모드에서):**
  * `i`: 커서 **앞(Insert)** 부터 텍스트 입력 시작
  * `a`: 커서 **뒤(Append)** 부터 텍스트 입력 시작
  * `o`: 현재 줄 **아래(Open)** 에 새 빈 줄을 만들고 입력 모드 시작
  * `O`: 현재 줄 **위(Open)** 에 새 빈 줄을 만들고 입력 모드 시작

* **일반 모드로 복귀:** 입력 모드에서 `Esc` 키를 누릅니다.

* **저장 및 종료 (명령-라인 모드에서):**
    1. `Esc` 키를 눌러 **일반 모드**인지 확인합니다.
    2. 콜론(`:`) 키를 누르면 화면 하단에 `:` 가 나타납니다.
    3. 원하는 명령어를 입력하고 Enter 키를 누릅니다.
        * `:w`: 현재 파일 **저장 (Write)**
        * `:q`: 편집기 **종료 (Quit)** (변경 사항이 있는데 저장하지 않았으면 에러 발생)
        * `:wq`: **저장하고 종료**
        * `:x`: 저장하고 종료 (`:wq` 와 유사하지만 파일 변경이 있을 때만 저장)
        * `ZZ`: (일반 모드에서) 저장하고 종료 (Shift + Z 두 번)
        * `:q!`: 변경 사항을 **저장하지 않고 강제 종료**. **주의해서 사용!**
        * `:w <새 파일 이름>`: 현재 내용을 지정한 새 파일 이름으로 저장 (다른 이름으로 저장)

"Vim을 종료할 줄 몰라서 재부팅했다"는 농담이 있을 정도로 처음에는 종료 방법이 헷갈릴 수 있습니다. **`:wq`** 또는 **`:q!`** 를 기억하세요!


### 5.2.3 필수 편집 기능 (Essential Editing)

텍스트를 삭제하고, 복사하고, 붙여넣는 기본적인 편집 기능입니다. **모두 일반 모드(Normal Mode)에서 실행합니다.**

* **삭제 (Deleting):**
  * `x`: 커서 위치의 한 글자 삭제
  * `dw`: 커서 위치부터 현재 단어 끝까지 삭제 (**D**elete **W**ord)
  * `dd`: 현재 줄 전체 삭제 (**D**elete Line)
  * `d$`: 커서 위치부터 현재 줄 끝까지 삭제 (축약형 `D`도 가능)
  * `<숫자>dd`: 현재 줄부터 아래로 `<숫자>`만큼의 줄 삭제 (예: `5dd` -> 5줄 삭제)

* **복사 (Copying = Yanking):** Vim에서는 복사를 'Yank'라고 부릅니다.
  * `yw`: 커서 위치부터 현재 단어 끝까지 복사 (**Y**ank **W**ord)
  * `yy`: 현재 줄 전체 복사 (**Y**ank Line)
  * `<숫자>yy`: 현재 줄부터 아래로 `<숫자>`만큼의 줄 복사 (예: `3yy` -> 3줄 복사)

* **붙여넣기 (Pasting):**
  * `p`: 복사하거나 삭제(`d`, `x` 로 삭제된 내용도 임시 저장됨)한 내용을 커서 **다음 위치**에 붙여넣기 (**P**aste)
  * `P`: 커서 **이전 위치**에 붙여넣기

* **실행 취소 / 다시 실행 (Undo / Redo):**
  * `u`: 마지막 명령 실행 **취소 (Undo)**
  * `Ctrl + r`: 마지막에 취소했던 명령 **다시 실행 (Redo)**

* **검색 (Searching - 명령-라인 모드):**
    1. 일반 모드에서 슬래시(`/`) 키를 누릅니다.
    2. 찾고 싶은 단어(패턴)를 입력하고 Enter 키를 누릅니다.
    3. `n` 키를 누르면 다음 일치 항목으로 이동합니다.
    4. `N` 키를 누르면 이전 일치 항목으로 이동합니다.
  * 물음표(`?`)를 사용하면 위쪽(역방향)으로 검색합니다 (`?<패턴>`).

* **치환 (Replacing - 명령-라인 모드):**
  * `:%s/<찾을 내용>/<바꿀 내용>/g`
    * `%`: 파일 전체 범위
    * `s`: 치환(substitute) 명령
    * `<찾을 내용>`: 검색할 문자열 또는 정규 표현식
    * `<바꿀 내용>`: 변경할 문자열
    * `g`: (global) 현재 행에서 여러 번 일치해도 모두 변경. 없으면 각 행에서 첫 번째 일치 항목만 변경.
  * `:%s/<찾을 내용>/<바꿀 내용>/gc`
    * `c`: (confirm) 변경할 때마다 사용자에게 확인을 받습니다. (y/n/a/q 등으로 응답)


### 5.2.4 간단한 Vim 설정 (`~/.vimrc`)

Vim의 다양한 동작(색상, 들여쓰기, 줄 번호 표시 등)은 설정 파일을 통해 사용자가 원하는 대로 변경할 수 있습니다. 이 설정 파일은 사용자의 홈 디렉토리에 있는 `.vimrc` 라는 숨김 파일입니다. (없으면 직접 만들면 됩니다.)

* **설정 파일 편집:** `vim ~/.vimrc`
* **유용한 설정 예시 (파일에 아래 내용 추가):**

    ```vimrc
    syntax on         " 문법 강조 기능 켜기
    set number        " 줄 번호 표시하기
    set cursorline    " 현재 커서가 있는 줄 강조하기
    set ignorecase    " 검색 시 대소문자 무시
    set smartcase     " 검색어에 대문자가 포함되면 대소문자 구분, 모두 소문자면 무시
    set autoindent    " 자동 들여쓰기
    set tabstop=4     " 탭 키 간격을 4칸으로 설정
    set shiftwidth=4  " 자동 들여쓰기 시 4칸으로 설정
    set expandtab     " 탭 키 입력 시 공백 문자로 변환
    set hlsearch      " 검색 결과 강조 표시
    set incsearch     " 입력 중에도 검색 결과 미리 보여주기
    ```

* **설정 적용:** `.vimrc` 파일을 저장하고 Vim을 다시 시작하거나, Vim 내에서 `:so %` 명령을 실행하면 변경된 설정이 적용됩니다.


### Vim 학습을 위한 팁: `vimtutor`

Vim 자체에 내장된 훌륭한 튜토리얼 프로그램이 있습니다. 터미널에 `vimtutor` 라고 입력하고 실행해보세요. 약 30분 정도 소요되는 대화형 튜토리얼을 통해 Vim의 기본적인 사용법을 직접 실습하며 익힐 수 있습니다. **Vim을 처음 배우는 분들께 강력히 추천합니다!**


## 5.3 쉬운 편집기 Nano (간단하고 직관적인 사용법) (Easy Editor Nano: Simple and Intuitive Usage)

Vim의 모드 개념이 너무 복잡하게 느껴지거나, 당장 간단하게 파일을 편집할 방법이 필요하다면 **Nano(나노)** 가 좋은 대안이 될 수 있습니다. Nano는 Vim과 달리 **모드 개념이 없으며**, 화면 하단에 주요 기능의 단축키가 항상 표시되어 있어 배우기 쉽고 직관적입니다.

많은 리눅스 배포판에 기본적으로 포함되어 있거나, 없다면 쉽게 설치할 수 있습니다. (예: `sudo apt install nano`, `sudo dnf install nano`)


### 5.3.1 실행 및 기본 인터페이스 (Starting and Basic Interface)

* **Nano 시작하기:** `nano <파일 이름>` (기존 파일을 열거나 새 파일을 만듭니다.)
* **인터페이스:**
  * 화면 상단: 편집 중인 파일 이름이 표시됩니다.
  * 화면 중간: 텍스트를 입력하고 편집하는 주 영역입니다.
  * 화면 하단: `^X Exit`, `^O WriteOut` 처럼 주요 기능과 해당 단축키가 두 줄로 표시됩니다. 여기서 `^` 기호는 `Ctrl` 키를 의미합니다. 즉, `^X` 는 `Ctrl + X` 를 누르라는 뜻입니다.


### 5.3.2 파일 열기, 편집, 저장, 종료 (Opening, Editing, Saving, Quitting)

Nano는 우리가 흔히 사용하는 메모장 같은 그래픽 편집기와 사용법이 매우 유사합니다.

* **텍스트 편집:** 별도의 모드 전환 없이 바로 키보드로 텍스트를 입력하고 수정할 수 있습니다. 방향키로 커서를 이동하고, `Delete` 나 `Backspace` 키로 문자를 지울 수 있습니다.
* **파일 저장:** `Ctrl + O` (**O**utput) 키를 누릅니다. 화면 하단에 저장할 파일 이름을 묻는 프롬프트가 나타납니다.
  * 현재 이름 그대로 저장하려면 그냥 `Enter` 키를 누릅니다.
  * 다른 이름으로 저장하고 싶다면 새 파일 이름을 입력하고 `Enter` 키를 누릅니다.
* **Nano 종료:** `Ctrl + X` (E**x**it) 키를 누릅니다.
  * 만약 변경 사항이 있는데 저장하지 않았다면, 저장할 것인지 묻는 메시지가 나타납니다.
    * `Y` 키를 누르면 저장 과정을 진행합니다 (파일 이름 확인).
    * `N` 키를 누르면 저장하지 않고 그냥 종료합니다.
    * `Ctrl + C` 키를 누르면 종료를 취소하고 편집 화면으로 돌아갑니다.


### 5.3.3 주요 단축키 안내 (Key Shortcuts)

Nano 화면 하단에 항상 표시되는 단축키들이 가장 중요합니다. 몇 가지 추가로 유용한 단축키는 다음과 같습니다. (전체 목록은 `Ctrl + G` 로 확인)

* `Ctrl + G`: 도움말 화면 보기 (다시 `Ctrl + X` 로 닫기)
* `Ctrl + O`: 파일 저장 (Write **O**ut)
* `Ctrl + X`: Nano 종료 (E**x**it)
* `Ctrl + W`: 텍스트 검색 (**W**here Is)
* `Ctrl + K`: 현재 커서가 있는 줄 잘라내기 (**K**ut)
* `Ctrl + U`: 마지막으로 잘라낸 내용 붙여넣기 (**U**ncut)
* `Ctrl + C`: 현재 커서 위치 정보(줄 번호, 칸 번호) 표시 (**C**urrent Position)
* `Alt + A` (또는 `Ctrl + 6`): 텍스트 선택 시작점 표시 (Mark Set). 이후 방향키로 텍스트를 선택할 수 있습니다.
  * 선택된 상태에서 `Alt + 6`: 선택 영역 복사 (Copy Text)
  * 선택된 상태에서 `Ctrl + K`: 선택 영역 잘라내기

Nano는 Vim처럼 복잡한 기능은 없지만, 간단한 설정 파일 수정이나 스크립트 작성 등 기본적인 편집 작업에는 충분히 편리하고 쉽습니다.


## 결론: 나에게 맞는 편집기는?

이번 챕터에서는 터미널 환경에서의 필수 생존 기술인 텍스트 편집기 사용법을 배웠습니다.

* **Vim:** 강력하고 효율적이지만, 모드 개념 때문에 초기 학습 곡선이 가파릅니다. 하지만 한번 익숙해지면 키보드만으로 매우 빠르게 작업할 수 있으며, 거의 모든 리눅스/유닉스 환경에서 사용할 수 있다는 큰 장점이 있습니다. (`:w`, `:q`, `:wq`, `:q!`, `i`, `Esc`, `dd`, `yy`, `p`, `/` 등 기본 생존 명령어를 기억하세요!)
* **Nano:** 모드 개념 없이 직관적이고 배우기 쉽습니다. 화면 하단에 단축키가 항상 안내되므로 초보자가 사용하기 편리합니다. (`Ctrl + O`, `Ctrl + X` 가 가장 중요!)

**추천:** 두 편집기 모두 사용해보는 것이 좋습니다! 당장 급하게 파일을 수정해야 하거나 Vim이 너무 어렵게 느껴진다면 **Nano**로 시작하세요. 하지만 장기적인 효율성과 리눅스 환경 적응력을 위해서는 시간을 투자하여 **Vim의 기초를 익혀두는 것**을 강력히 권장합니다. (`vimtutor`를 꼭 실행해보세요!) 최소한 Vim에서 파일을 열고, 간단히 수정하고, 저장하고 나올 수 있는 능력은 어떤 상황에서도 도움이 될 것입니다.
