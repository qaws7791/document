---
title: "3. 필수 명령어 (Essential Commands)"
---

리눅스를 사용하면서 가장 기본적이고 자주 사용하게 되는 필수 명령어들을 소개합니다. 이 명령어들을 숙지하면 파일 시스템을 탐색하고 관리하며, 텍스트 파일을 다루고 시스템 정보를 확인하는 등 기본적인 작업을 수행할 수 있습니다.


## 3.1. 디렉토리 탐색 및 기본 조작

파일 시스템을 효과적으로 탐색하고 관리하기 위한 기본적인 명령어들입니다.


### 3.1.1. pwd, cd, ls (옵션 포함: -l, -a, -h)

* **`pwd` (Print Working Directory):** 현재 자신이 위치한 디렉토리의 절대 경로를 화면에 출력합니다. 파일 시스템 내에서 현재 작업 중인 위치를 확인하는 데 사용됩니다.

    ```bash
    pwd
    ```

    출력 예시:

    ```
    /home/user
    ```

* **`cd` (Change Directory):** 현재 작업 디렉토리를 다른 디렉토리로 변경합니다.

    ```bash
    cd <이동할 디렉토리 경로>
    ```

  * `cd`: 홈 디렉토리로 이동합니다.
  * `cd ..`: 현재 디렉토리의 부모 디렉토리로 이동합니다.
  * `cd -`: 이전 작업 디렉토리로 이동합니다.
  * `cd /path/to/directory`: 절대 경로를 사용하여 특정 디렉토리로 이동합니다.
  * `cd relative/path`: 현재 디렉토리를 기준으로 하는 상대 경로를 사용하여 이동합니다.

* **`ls` (List):** 현재 디렉토리 또는 지정된 디렉토리의 파일 및 하위 디렉토리 목록을 화면에 출력합니다. 다양한 옵션을 사용하여 출력 방식을 제어할 수 있습니다.

    ```bash
    ls [옵션] [디렉토리 경로]
    ```

  * **`-l` (long listing):** 파일의 상세 정보를 함께 출력합니다. 권한, 소유자, 그룹, 파일 크기, 수정 시간, 파일 이름 등을 확인할 수 있습니다.

        ```bash
        ls -l
        ```

        출력 예시:

        ```
        -rw-r--r-- 1 user group  123 Apr  6 07:04 file.txt
        drwxr-xr-- 2 user group 4096 Apr  6 07:04 directory
        ```

  * **`-a` (all):** 숨겨진 파일 및 디렉토리(`.`으로 시작하는 이름)를 포함하여 모든 항목을 출력합니다.

        ```bash
        ls -a
        ```

        출력 예시:

        ```
        .  ..  .hidden_file  directory  file.txt
        ```

  * **`-h` (human-readable):** 파일 크기를 사람이 읽기 쉬운 단위 (K, M, G 등)로 표시합니다. `-l` 옵션과 함께 자주 사용됩니다.

        ```bash
        ls -lh
        ```

        출력 예시:

        ```
        -rw-r--r-- 1 user group  123 Apr  6 07:04 file.txt
        drwxr-xr-- 2 user group 4.0K Apr  6 07:04 directory
        ```

  * **`-al` 또는 `-la`:** `-a`와 `-l` 옵션을 함께 사용하여 숨겨진 파일を含む 상세 정보를 출력합니다.

        ```bash
        ls -al
        ```


### 3.1.2. mkdir, rmdir

* **`mkdir` (Make Directory):** 새로운 디렉토리를 생성합니다.

    ```bash
    mkdir <생성할 디렉토리 이름>
    ```

  * `mkdir -p <경로>`: 지정된 경로에 필요한 부모 디렉토리가 없는 경우에도 함께 생성합니다.

        ```bash
        mkdir mydir
        mkdir -p path/to/new/directory
        ```

* **`rmdir` (Remove Directory):** 비어 있는 디렉토리를 삭제합니다. 디렉토리 안에 파일이나 하위 디렉토리가 존재하면 삭제할 수 없습니다.

    ```bash
    rmdir <삭제할 디렉토리 이름>
    ```

    비어 있지 않은 디렉토리를 삭제하거나, 파일까지 함께 삭제하려면 `rm` 명령어를 사용해야 합니다 (3.2.2. `cp, mv, rm` 섹션 참고).

    ```bash
    rmdir empty_dir
    ```


## 3.2. 파일 관리

파일을 생성, 확인, 복사, 이동, 삭제하는 등 기본적인 파일 관리 명령어들입니다.


### 3.2.1. touch, cat, less, more, head, tail

* **`touch`**: 파일의 접근 및 수정 시간을 현재 시간으로 갱신하거나, 내용이 없는 빈 파일을 새로 생성합니다.

    ```bash
    touch <파일 이름>
    ```

  * 존재하지 않는 파일을 지정하면 해당 이름의 빈 파일이 생성됩니다.
  * 이미 존재하는 파일을 지정하면 파일의 최종 수정 시간이 현재 시간으로 업데이트됩니다.

* **`cat` (concatenate)**: 파일의 내용을 화면에 출력합니다. 짧은 파일 내용을 빠르게 확인하는 데 유용합니다. 여러 파일을 인수로 지정하면 파일 내용을 순서대로 연결하여 출력합니다.

    ```bash
    cat <파일 이름>
    ```

  * `cat file1.txt file2.txt`: `file1.txt` 내용 뒤에 `file2.txt` 내용을 이어서 출력합니다.

* **`less`**: 파일 내용을 한 화면씩 넘겨가며 볼 수 있는 프로그램입니다. 긴 파일 내용을 확인하는 데 유용하며, 다양한 탐색 기능을 제공합니다 (예: 검색, 줄 번호 표시 등).

    ```bash
    less <파일 이름>
    ```

  * 화면 이동: `Space` (다음 페이지), `b` (이전 페이지)
  * 검색: `/` + 검색어 + `Enter` (다음 검색 결과: `n`, 이전 검색 결과: `N`)
  * 종료: `q`

* **`more`**: `less`와 유사하게 파일 내용을 화면 단위로 출력하지만, 기능은 더 제한적입니다 (이전 페이지 이동 불가 등).

    ```bash
    more <파일 이름>
    ```

  * 화면 이동: `Space` (다음 페이지)
  * 종료: `q`

* **`head`**: 파일의 처음 몇 줄을 화면에 출력합니다. 기본적으로 처음 10줄을 보여줍니다.

    ```bash
    head <파일 이름>
    ```

  * `-n <숫자>` 옵션을 사용하여 출력할 줄 수를 지정할 수 있습니다.

        ```bash
        head -n 20 <파일 이름>  # 처음 20줄 출력
        ```

* **`tail`**: 파일의 마지막 몇 줄을 화면에 출력합니다. 로그 파일의 최근 내용을 확인하는 데 유용하며, `-f` 옵션을 사용하여 파일 내용이 실시간으로 추가되는 것을 감시할 수 있습니다. 기본적으로 마지막 10줄을 보여줍니다.

    ```bash
    tail <파일 이름>
    ```

  * `-n <숫자>` 옵션을 사용하여 출력할 줄 수를 지정할 수 있습니다.

        ```bash
        tail -n 5 <파일 이름>   # 마지막 5줄 출력
        ```

  * `-f` (follow): 파일 내용이 추가될 때마다 실시간으로 내용을 화면에 출력합니다. 로그 파일 감시에 유용하며, 종료하려면 `Ctrl` + `C`를 누릅니다.

        ```bash
        tail -f <로그 파일 이름>
        ```


### 3.2.2. cp, mv, rm (옵션 포함: -r, -f, -i)

* **`cp` (copy)**: 파일 또는 디렉토리를 복사합니다.

    ```bash
    cp <원본 파일> <대상 파일 또는 디렉토리>
    cp <원본 파일1> <원본 파일2> ... <대상 디렉토리>
    ```

  * `<대상>`이 디렉토리인 경우, 원본 파일들이 해당 디렉토리 아래에 같은 이름으로 복사됩니다.
  * `<대상>`이 파일인 경우, 원본 파일의 내용으로 덮어쓰거나 새로운 이름으로 복사됩니다.

  * **`-r` (recursive)**: 디렉토리를 복사할 때 하위 디렉토리와 파일들을 포함하여 재귀적으로 복사합니다. 디렉토리를 복사할 때는 반드시 이 옵션을 사용해야 합니다.

        ```bash
        cp -r <원본 디렉토리> <대상 디렉토리>
        ```

  * **`-f` (force)**: 대상 파일이 이미 존재하는 경우, 확인 없이 강제로 덮어씁니다.

        ```bash
        cp -f <원본 파일> <대상 파일>
        ```

  * **`-i` (interactive)**: 대상 파일이 이미 존재하는 경우, 덮어쓸 것인지 사용자에게 확인 메시지를 표시합니다.

        ```bash
        cp -i <원본 파일> <대상 파일>
        ```

* **`mv` (move)**: 파일 또는 디렉토리의 이름을 변경하거나 위치를 이동합니다.

    ```bash
    mv <원본 파일> <새로운 파일 이름 또는 대상 디렉토리>
    mv <원본 파일1> <원본 파일2> ... <대상 디렉토리>
    mv <원본 디렉토리> <새로운 디렉토리 이름 또는 대상 디렉토리>
    ```

  * `<대상>`이 존재하는 디렉토리인 경우, 원본 파일/디렉토리가 해당 디렉토리로 이동됩니다.
  * `<대상>`이 존재하지 않는 경우, 원본 파일/디렉토리의 이름이 `<대상>`으로 변경됩니다.

  * **`-f` (force)**: 대상 파일이 이미 존재하는 경우, 확인 없이 강제로 덮어씁니다.

        ```bash
        mv -f <원본 파일> <대상 파일>
        ```

  * **`-i` (interactive)**: 대상 파일이 이미 존재하는 경우, 덮어쓸 것인지 사용자에게 확인 메시지를 표시합니다.

        ```bash
        mv -i <원본 파일> <대상 파일>
        ```

* **`rm` (remove)**: 파일 또는 디렉토리를 삭제합니다. **주의해서 사용해야 하며, 삭제된 파일은 복구하기 어려울 수 있습니다.**

    ```bash
    rm <파일 이름>
    rm <파일 이름1> <파일 이름2> ...
    ```

  * **`-r` (recursive)**: 디렉토리와 그 안의 모든 하위 디렉토리 및 파일을 재귀적으로 삭제합니다. 디렉토리를 삭제할 때는 이 옵션을 사용해야 합니다.

        ```bash
        rm -r <디렉토리 이름>
        ```

  * **`-f` (force)**: 파일이 존재하지 않아도 오류 메시지를 표시하지 않고, 쓰기 금지된 파일을 강제로 삭제합니다 (권한이 있는 경우). 매우 주의해서 사용해야 합니다.

        ```bash
        rm -f <파일 이름>
        rm -rf <디렉토리 이름>  # 디렉토리 및 내용 강제 삭제 (매우 위험)
        ```

  * **`-i` (interactive)**: 삭제할 때마다 사용자에게 확인 메시지를 표시합니다. 실수로 파일을 삭제하는 것을 방지하는 데 유용합니다.

        ```bash
        rm -i <파일 이름>
        rm -ri <디렉토리 이름>
        ```


## 3.3. 텍스트 처리 및 검색

텍스트 파일의 내용을 검색하고 조작하는 데 사용되는 필수 명령어들입니다.


### 3.3.1. grep, find, locate

* **`grep` (Global Regular Expression Print)**: 하나 이상의 입력 파일에서 특정 패턴과 일치하는 줄을 검색하여 출력합니다. 정규 표현식을 사용하여 더욱 복잡한 패턴 검색이 가능합니다.

    ```bash
    grep [옵션] <패턴> [파일...]
    ```

  * `<패턴>`: 검색할 문자열 또는 정규 표현식입니다.
  * `[파일...]`: 검색할 파일 목록입니다. 파일 이름을 생략하면 표준 입력에서 검색합니다.

  * **`-i` (ignore-case)**: 대소문자를 구분하지 않고 검색합니다.

        ```bash
        grep -i "linux" file.txt
        ```

  * **`-n` (line-number)**: 일치하는 줄과 함께 줄 번호를 출력합니다.

        ```bash
        grep -n "error" logfile.log
        ```

  * **`-r` 또는 `-R` (recursive)**: 지정된 디렉토리와 그 하위 디렉토리의 모든 파일에서 패턴을 검색합니다.

        ```bash
        grep -r "function" ./src
        ```

  * **`-v` (invert-match)**: 지정된 패턴과 일치하지 않는 줄만 출력합니다.

        ```bash
        grep -v "comment" code.py
        ```

  * **`-c` (count)**: 패턴과 일치하는 줄의 총 개수를 출력합니다.

        ```bash
        grep -c "warning" system.log
        ```

* **`find`**: 지정된 디렉토리에서 다양한 조건을 사용하여 파일을 검색합니다. 파일 이름, 크기, 수정 시간, 권한 등 다양한 속성을 기준으로 검색할 수 있으며, 찾은 파일에 대해 특정 명령을 실행할 수도 있습니다.

    ```bash
    find <경로> [옵션] [표현식]
    ```

  * `<경로>`: 검색을 시작할 디렉토리입니다. `.`은 현재 디렉토리를 의미합니다.
  * `[옵션]`: 검색 동작 방식을 제어하는 옵션입니다.
  * `[표현식]`: 검색 조건을 지정합니다 (예: `-name <이름 패턴>`, `-type f` (파일), `-type d` (디렉토리), `-size +10M` (10MB보다 큰 파일), `-mtime -7` (최근 7일 이내에 수정된 파일)).

  * **`-name <패턴>`**: 지정된 파일 이름 패턴과 일치하는 파일을 찾습니다 (와일드카드 `*`, `?` 사용 가능).

        ```bash
        find . -name "*.txt"
        find /home/user -name "report*"
        ```

  * **`-type <f|d|l>`**: 파일 유형별로 검색합니다 (`f`: 파일, `d`: 디렉토리, `l`: 심볼릭 링크).

        ```bash
        find / -type d -name "apache2"
        ```

  * **`-size <+/-n[ckMG]>`**: 파일 크기를 기준으로 검색합니다 (`+n`: n보다 큼, `-n`: n보다 작음, `n`: 정확히 n, `c`: 바이트, `k`: 킬로바이트, `M`: 메가바이트, `G`: 기가바이트).

        ```bash
        find /var/log -size +10M
        ```

  * **`-mtime <+/-n>`**: 파일 수정 시간을 기준으로 검색합니다 (`+n`: n일 이전, `-n`: n일 이내, `n`: 정확히 n일 전).

        ```bash
        find /home/user -mtime -30
        ```

  * **`-exec <명령> {} \;`**: 찾은 파일에 대해 지정된 명령을 실행합니다 (`{}`는 찾은 파일을 나타내는 플레이스홀더, `\;`는 명령 종료를 나타냅니다).

        ```bash
        find . -name "*.log" -exec rm {} \;  # 확장자가 .log인 파일 삭제 (주의!)
        find . -type f -name "*.txt" -exec grep "keyword" {} \; -print # .txt 파일에서 "keyword" 검색 후 파일 이름 출력
        ```

* **`locate`**: 미리 구축된 파일 이름 데이터베이스를 사용하여 빠르게 파일을 검색합니다. `find`보다 빠르지만, 데이터베이스가 최신 상태가 아닐 수 있으므로 최근에 생성된 파일은 검색되지 않을 수 있습니다. `updatedb` 명령어를 사용하여 데이터베이스를 업데이트할 수 있습니다 (관리자 권한 필요).

    ```bash
    locate [옵션] <패턴>
    ```

  * `<패턴>`: 검색할 파일 이름 패턴입니다 (와일드카드 사용 가능).

  * **`-i`**: 대소문자를 구분하지 않고 검색합니다.

        ```bash
        locate -i "report.pdf"
        ```

  * **`-n <숫자>`**: 최대 출력 결과 수를 제한합니다.

        ```bash
        locate -n 10 "image"
        ```


### 3.3.2. 파이프, 리다이렉션

리눅스 셸은 파이프와 리다이렉션을 통해 명령어의 입출력을 유연하게 연결하고 관리할 수 있는 강력한 기능을 제공합니다.

* **파이프 (`|`)**: 한 명령어의 표준 출력을 다른 명령어의 표준 입력으로 연결합니다. 여러 명령어를 조합하여 복잡한 작업을 수행할 수 있습니다.

    ```bash
    명령어1 | 명령어2 | 명령어3 ...
    ```

    예시: 현재 실행 중인 프로세스 중에서 "apache"를 포함하는 프로세스 목록을 확인합니다.

    ```bash
    ps aux | grep "apache"
    ```

    `ps aux` 명령어의 출력 결과가 `grep "apache"` 명령어의 입력으로 전달되어, "apache"라는 문자열을 포함하는 줄만 화면에 출력됩니다.

* **리다이렉션 (`>`, `>>`, `<`)**: 명령어의 표준 입력, 표준 출력, 표준 에러 스트림의 방향을 변경합니다.

  * **`>` (출력 리다이렉션)**: 명령어의 표준 출력을 지정된 파일로 덮어씁니다. 파일이 존재하지 않으면 새로 생성합니다.

        ```bash
        ls -l > file_list.txt
        ```

        `ls -l` 명령어의 출력 결과가 `file_list.txt` 파일에 저장됩니다. 파일 내용이 이미 존재하면 덮어쓰므로 주의해야 합니다.

  * **`>>` (출력 추가 리다이렉션)**: 명령어의 표준 출력을 지정된 파일의 끝에 추가합니다. 파일이 존재하지 않으면 새로 생성합니다.

        ```bash
        echo "New log entry" >> server.log
        ```

        `echo` 명령어의 출력 결과가 `server.log` 파일의 마지막 줄에 추가됩니다.

  * **`<` (입력 리다이렉션)**: 명령어의 표준 입력을 지정된 파일로부터 읽어옵니다.

        ```bash
        mail -s "Report" < report_content.txt
        ```

        `mail` 명령어의 본문 내용이 `report_content.txt` 파일로부터 읽어와져 메일이 발송됩니다.

파이프와 리다이렉션은 리눅스 환경에서 데이터를 처리하고 관리하는 데 매우 유용한 도구이며, 다양한 명령어를 조합하여 강력한 기능을 발휘할 수 있습니다.


## 3.4. 기본 텍스트 편집기 사용법

리눅스 환경에서는 명령행에서 텍스트 파일을 편집할 수 있는 여러 텍스트 편집기가 제공됩니다. 초보자에게 추천되는 `nano`와 기본적인 사용법을 알아두면 유용한 `vim` / `vi`의 기본 사용법을 소개합니다.


### 3.4.1. nano 사용법 (초보자 추천)

`nano`는 명령행 기반의 간단하고 사용하기 쉬운 텍스트 편집기입니다. 화면 하단에 주요 명령어들이 표시되어 초보자도 쉽게 사용할 수 있습니다.

**실행:**

터미널에서 `nano` 명령어 뒤에 편집할 파일 이름을 지정하여 실행합니다. 파일이 없으면 새로운 파일을 생성합니다.

```bash
nano <파일 이름>
```

**기본 조작:**

* **커서 이동:** 화살표 키를 사용하여 텍스트 내에서 커서를 이동합니다.
* **텍스트 입력:** 키보드를 사용하여 텍스트를 입력합니다.
* **주요 명령어 (화면 하단 표시):**
  * `^G` 또는 `Ctrl` + `G`: 도움말 보기
  * `^O` 또는 `Ctrl` + `O`: 저장하기
    * 파일 이름을 변경하고 싶으면 이 단계에서 새로운 이름을 입력하고 Enter 키를 누릅니다.
  * `^R` 또는 `Ctrl` + `R`: 파일 읽어오기 (현재 커서 위치에 다른 파일 내용 삽입)
  * `^Y` 또는 `Ctrl` + `Y`: 이전 페이지
  * `^V` 또는 `Ctrl` + `V`: 다음 페이지
  * `^K` 또는 `Ctrl` + `K`: 현재 줄 잘라내기
  * `^U` 또는 `Ctrl` + `U`: 잘라낸 줄 붙여넣기 (현재 커서 위치에)
  * `^W` 또는 `Ctrl` + `W`: 검색
  * `^\` 또는 `Ctrl` + `\`: 바꾸기
  * `^_` 또는 `Ctrl` + `_`: 특정 줄/열로 이동
  * `^X` 또는 `Ctrl` + `X`: 종료하기
    * 변경 사항이 있으면 저장할지 여부를 묻는 메시지가 나타납니다. `Y` (Yes), `N` (No), `C` (Cancel) 중 하나를 선택합니다.

**예시:**

`my_document.txt` 파일을 `nano`로 열고 내용을 편집한 후 저장하고 종료하는 과정:

1. 터미널에 `nano my_document.txt` 입력 후 Enter 키를 누릅니다.
2. 텍스트를 입력하거나 기존 내용을 수정합니다.
3. 저장하려면 `Ctrl` + `O`를 누르고 Enter 키를 누릅니다 (기본 파일 이름 유지).
4. 종료하려면 `Ctrl` + `X`를 누릅니다.


### 3.4.2. vim / vi 기본 사용법 (이동, 입력, 저장, 종료)

`vim` (Vi IMproved)은 강력하고 다양한 기능을 제공하는 텍스트 편집기입니다. `vi`는 `vim`의 원조이며, 대부분의 리눅스 시스템에 기본적으로 설치되어 있습니다. `vim`은 모드 기반 편집 방식을 사용하므로 처음에는 다소 어렵게 느껴질 수 있지만, 익숙해지면 매우 효율적으로 텍스트를 편집할 수 있습니다.

**실행:**

터미널에서 `vim` 또는 `vi` 명령어 뒤에 편집할 파일 이름을 지정하여 실행합니다.

```bash
vim <파일 이름>
vi <파일 이름>
```

**기본 모드:**

`vim`을 실행하면 기본적으로 **명령 모드 (Command Mode)** 상태가 됩니다. 이 모드에서는 텍스트를 입력하는 대신 편집 명령어를 실행할 수 있습니다.

**입력 모드 (Insert Mode):**

텍스트를 입력하려면 명령 모드에서 특정 키를 눌러 입력 모드로 전환해야 합니다.

* `i`: 현재 커서 위치에서 입력 모드로 전환합니다.
* `a`: 현재 커서 위치 다음에서 입력 모드로 전환합니다.
* `o`: 현재 줄 아래에 새로운 줄을 삽입하고 입력 모드로 전환합니다.
* `I`: 현재 줄의 처음으로 이동하여 입력 모드로 전환합니다.
* `A`: 현재 줄의 끝으로 이동하여 입력 모드로 전환합니다.
* `O`: 현재 줄 위에 새로운 줄을 삽입하고 입력 모드로 전환합니다.

입력 모드에서는 일반적인 텍스트 편집기와 같이 키보드를 사용하여 텍스트를 입력할 수 있습니다. 입력 모드에서 명령 모드로 돌아가려면 `Esc` 키를 누릅니다.

**커서 이동 (명령 모드):**

명령 모드에서 다음 키들을 사용하여 커서를 이동할 수 있습니다.

* `h`: 왼쪽으로 한 문자 이동
* `j`: 아래로 한 줄 이동
* `k`: 위로 한 줄 이동
* `l`: 오른쪽으로 한 문자 이동
* `w`: 다음 단어의 시작으로 이동
* `b`: 이전 단어의 시작으로 이동
* `gg`: 파일의 처음 줄로 이동
* `G`: 파일의 마지막 줄로 이동
* `:n`: n번째 줄로 이동 (예: `:10`은 10번째 줄로 이동)

**저장 및 종료 (명령 모드):**

명령 모드에서 다음 명령어를 입력하고 Enter 키를 눌러 저장 및 종료를 수행합니다 (명령어는 `:`으로 시작합니다).

* `:w`: 현재 파일을 저장합니다.
* `:wq` 또는 `:x`: 현재 파일을 저장하고 종료합니다.
* `:q`: 변경 사항이 없으면 종료합니다.
* `:q!`: 변경 사항을 저장하지 않고 강제로 종료합니다 (주의!).

**기본 편집 명령어 (명령 모드):**

* `dd`: 현재 줄을 잘라내기 (삭제) 합니다.
* `yy`: 현재 줄을 복사 (yank) 합니다.
* `p`: 잘라내거나 복사한 내용을 현재 커서 아래에 붙여넣기 (paste) 합니다.
* `P`: 잘라내거나 복사한 내용을 현재 커서 위에 붙여넣기 합니다.
* `u`: 마지막 변경 사항을 취소 (undo) 합니다.
* `Ctrl` + `r`: 취소된 변경 사항을 다시 실행 (redo) 합니다.
* `/` + 검색어 + `Enter`: 현재 커서 위치 이후에서 검색어를 찾습니다.
  * `n`: 다음 검색 결과로 이동
  * `N`: 이전 검색 결과로 이동

**예시:**

`my_document.txt` 파일을 `vim`으로 열고 내용을 편집한 후 저장하고 종료하는 기본적인 과정:

1. 터미널에 `vim my_document.txt` 입력 후 Enter 키를 누릅니다 (명령 모드).
2. 입력 모드로 전환하기 위해 `i` 키를 누릅니다.
3. 텍스트를 입력하거나 기존 내용을 수정합니다.
4. 명령 모드로 돌아가려면 `Esc` 키를 누릅니다.
5. 저장하고 종료하려면 `:` 키를 누른 후 `wq`를 입력하고 Enter 키를 누릅니다.

`vim`은 처음에는 학습 곡선이 있지만, 숙달되면 매우 강력하고 효율적인 텍스트 편집 도구입니다. 꾸준히 사용하면서 다양한 명령어를 익히는 것이 중요합니다.


## 3.5. 시스템 정보 확인

리눅스 시스템의 기본적인 정보 (커널 버전, 디스크 사용량, 메모리 상태, 실행 중인 프로세스 등)를 확인하는 데 사용되는 필수 명령어들입니다.


### 3.5.1. uname, df, du, free, top, htop

* **`uname` (Unix name)**: 시스템의 기본적인 정보를 출력합니다.

    ```bash
    uname [옵션]
    ```

  * `-a` (all): 모든 시스템 정보를 출력합니다 (커널 이름, 호스트 이름, 커널 릴리스, 커널 버전, 머신 아키텍처, 운영체제).

        ```bash
        uname -a
        ```

        출력 예시:

        ```
        Linux myhost 5.15.0-71-generic #78-Ubuntu SMP Tue Apr 18 09:02:29 UTC 2023 x86_64 x86_64 x86_64 GNU/Linux
        ```

  * `-s` (kernel-name): 커널 이름을 출력합니다 (일반적으로 "Linux").
  * `-n` (nodename): 호스트 이름을 출력합니다.
  * `-r` (kernel-release): 커널 릴리스 버전을 출력합니다.
  * `-v` (kernel-version): 커널 버전을 출력합니다.
  * `-m` (machine): 시스템의 머신 아키텍처를 출력합니다.
  * `-o` (operating-system): 운영체제 이름을 출력합니다 (일반적으로 "GNU/Linux").

* **`df` (Disk Free)**: 파일 시스템의 디스크 공간 사용량을 표시합니다.

    ```bash
    df [옵션] [파일 시스템 또는 디렉토리]
    ```

  * `-h` (human-readable): 디스크 공간을 사람이 읽기 쉬운 단위 (K, M, G 등)로 표시합니다.

        ```bash
        df -h
        ```

        출력 예시:

        ```
        Filesystem      Size  Used Avail Use% Mounted on
        udev            7.8G     0  7.8G   0% /dev
        tmpfs           1.6G  9.1M  1.6G   1% /run
        /dev/sda1       228G   29G  188G  14% /
        tmpfs           7.9G     0  7.9G   0% /dev/shm
        tmpfs           5.0M     0  5.0M   0% /run/lock
        /dev/sdb1       917G  345G  525G  40% /data
        tmpfs           1.6G   16K  1.6G   1% /run/user/1000
        ```

  * `-T` (print-type): 파일 시스템 유형을 함께 표시합니다.

        ```bash
        df -hT
        ```

* **`du` (Disk Usage)**: 파일 및 디렉토리의 디스크 공간 사용량을 표시합니다.

    ```bash
    du [옵션] [파일 또는 디렉토리]
    ```

  * `-h` (human-readable): 디스크 공간을 사람이 읽기 쉬운 단위로 표시합니다.
  * `-s` (summarize): 각 인수의 총 사용량만 표시합니다.
  * `-a` (all): 모든 파일 및 디렉토리의 사용량을 표시합니다.
  * `-d <n>` (depth): 보고할 최대 디렉토리 깊이를 설정합니다.

        ```bash
        du -sh /home/user   # /home/user 디렉토리의 총 사용량 표시
        du -h --max-depth=1 /var/log # /var/log 디렉토리의 하위 디렉토리별 사용량 표시
        du -ah .             # 현재 디렉토리의 모든 파일 및 디렉토리 사용량 표시
        ```

* **`free`**: 시스템의 메모리 사용량 및 스왑 공간 정보를 표시합니다.

    ```bash
    free [옵션]
    ```

  * `-h` (human-readable): 메모리 크기를 사람이 읽기 쉬운 단위로 표시합니다.
  * `-m` (megabytes): 메모리 크기를 메가바이트 단위로 표시합니다.
  * `-g` (gigabytes): 메모리 크기를 기가바이트 단위로 표시합니다.
  * `-s <n>` (seconds): n초마다 결과를 반복하여 표시합니다 (실시간 모니터링).
  * `-c <n>` (count): 결과를 n번만 표시합니다.

        ```bash
        free -h
        ```

        출력 예시:

        ```
                      total        used        free      shared  buff/cache   available
        Mem:           7.7G        1.5G        5.2G        113M        1.1G        5.9G
        Swap:          2.0G          0B        2.0G
        ```

    * `total`: 총 메모리 및 스왑 공간 크기
    * `used`: 현재 사용 중인 메모리 및 스왑 공간 크기
    * `free`: 현재 사용 가능한 메모리 및 스왑 공간 크기
    * `shared`: 공유 메모리 크기
    * `buff/cache`: 버퍼 및 캐시로 사용 중인 메모리 크기 (실제 사용 가능한 메모리에 포함될 수 있음)
    * `available`: 애플리케이션이 즉시 사용할 수 있는 메모리 크기 (free + buff/cache의 일부)

* **`top`**: 시스템의 실시간 프로세스 활동을 표시합니다. CPU 사용률, 메모리 사용률, 각 프로세스의 자원 사용량 등을 확인할 수 있습니다.

    ```bash
    top
    ```

    `top` 명령어를 실행하면 실시간으로 업데이트되는 정보 화면이 나타납니다. 주요 정보는 다음과 같습니다.

  * 시스템 요약 정보 (CPU 사용률, 메모리 사용률, 가동 시간 등)
  * 실행 중인 프로세스 목록 (PID, 사용자, CPU%, MEM%, COMMAND 등)

    `top` 명령어 실행 중 자주 사용되는 키:

  * `q`: 종료
  * `h`: 도움말
  * `k`: 특정 PID의 프로세스에 신호 보내기 (종료 등)
  * `M`: 메모리 사용량 순으로 정렬
  * `P`: CPU 사용률 순으로 정렬

* **`htop`**: `top`과 유사하지만, 더 사용자 친화적인 인터페이스와 추가 기능을 제공하는 실시간 프로세스 모니터링 도구입니다 (일반적으로 별도로 설치해야 합니다). 색상 표시, 마우스 조작, 프로세스 트리 보기, 특정 프로세스 검색 및 필터링 등의 기능을 제공합니다.

    ```bash
    htop
    ```

    `htop`은 `top`과 유사한 정보를 표시하지만, 시각적으로 더 보기 쉽고 사용하기 편리합니다. F1 ~ F10 키를 사용하여 다양한 기능을 이용할 수 있습니다.

이러한 시스템 정보 확인 명령어들은 시스템의 상태를 파악하고 성능 문제를 진단하는 데 필수적인 도구입니다.


## 3.6. 도움말 활용

리눅스 명령어는 다양한 옵션과 사용법을 가지고 있습니다. 각 명령어에 대한 자세한 정보를 얻는 것은 리눅스 사용 능력을 향상시키는 데 매우 중요합니다. 리눅스에서는 주로 `man`, `info`, `--help` 옵션을 통해 명령어 도움말을 확인할 수 있습니다.


### 3.6.1. man, info, --help 옵션

* **`man` (manual pages)**: 대부분의 리눅스 명령어 및 시스템 호출, 설정 파일 등에 대한 자세한 설명서를 제공합니다. 각 매뉴얼 페이지는 다양한 섹션으로 구성되어 있으며, 명령어의 이름, 사용법, 설명, 옵션, 관련 명령어 등을 포함합니다.

    ```bash
    man <명령어 이름>
    ```

    예시: `ls` 명령어에 대한 매뉴얼 페이지를 보려면 다음과 같이 입력합니다.

    ```bash
    man ls
    ```

    `man` 페이지 내에서의 이동:

  * 화살표 키 또는 `j` (아래로 한 줄), `k` (위로 한 줄)
  * `Space` (다음 페이지), `b` (이전 페이지)
  * `/` + 검색어 + `Enter` (검색)
  * `n` (다음 검색 결과), `N` (이전 검색 결과)
  * `q` (종료)

    매뉴얼 페이지는 일반적으로 다음과 같은 섹션으로 구성됩니다.

  * `NAME`: 명령어의 이름과 간단한 설명
  * `SYNOPSIS`: 명령어의 사용 형식
  * `DESCRIPTION`: 명령어에 대한 자세한 설명
  * `OPTIONS`: 명령어에서 사용할 수 있는 옵션 목록 및 설명
  * `EXAMPLES`: 명령어 사용 예시
  * `SEE ALSO`: 관련 명령어 및 매뉴얼 페이지

* **`info`**: GNU 프로젝트에서 제공하는 하이퍼텍스트 형식의 도움말 시스템입니다. `man` 페이지보다 더 구조화되고 연결된 정보를 제공할 수 있습니다.

    ```bash
    info <명령어 이름>
    ```

    예시: `grep` 명령어에 대한 `info` 페이지를 보려면 다음과 같이 입력합니다.

    ```bash
    info grep
    ```

    `info` 페이지 내에서의 이동:

  * 화살표 키 (이동)
  * `Space` (다음 페이지), `Backspace` (이전 페이지)
  * `n` (다음 노드), `p` (이전 노드), `u` (상위 노드)
  * `t` (제목으로 이동)
  * `l` (마지막 방문 노드로 이동)
  * `q` (종료)

    `info` 페이지는 여러 개의 "노드"로 구성되어 있으며, 탭 키를 사용하여 링크를 선택하고 Enter 키를 눌러 이동할 수 있습니다.

* **`--help` 옵션**: 많은 명령어들이 간단한 사용법과 주요 옵션 목록을 화면에 출력하는 `--help` 옵션을 지원합니다. `man` 페이지나 `info` 페이지보다 간결한 정보를 빠르게 확인하고 싶을 때 유용합니다.

    ```bash
    <명령어 이름> --help
    ```

    예시: `ls` 명령어의 간단한 도움말을 보려면 다음과 같이 입력합니다.

    ```bash
    ls --help
    ```

    출력 예시는 명령어에 따라 다르지만, 일반적으로 명령어의 기본적인 사용법과 자주 사용되는 옵션에 대한 간략한 설명을 제공합니다.

**어떤 도움말을 사용해야 할까요?**

* **`man`**: 가장 포괄적이고 자세한 정보를 제공합니다. 명령어의 모든 기능과 옵션에 대해 깊이 있게 이해하고 싶을 때 사용합니다.
* **`info`**: GNU 관련 명령어에 대한 구조화된 도움말을 제공합니다. `man` 페이지에서 찾기 어려운 추가 정보나 개념 설명을 포함할 수 있습니다.
* **`--help`**: 명령어의 기본적인 사용법과 주요 옵션을 빠르게 확인하고 싶을 때 사용합니다. 간단한 명령어의 사용법을 잊었거나, 자주 사용되는 옵션을 확인하고 싶을 때 유용합니다.

리눅스를 효과적으로 사용하기 위해서는 이러한 도움말 기능을 적극적으로 활용하는 습관을 들이는 것이 중요합니다. 궁금한 명령어의 사용법이 있다면 먼저 도움말을 찾아보는 것을 권장합니다.
