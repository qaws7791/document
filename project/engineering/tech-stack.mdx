## API Design

- REST - 가장 널리 사용하는 API로 리소스 기반의 주소를 지정하여 데이터를 자고 받는 방법

  [https://ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm](https://ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm)

- GraphQL - API를 위한 쿼리 언어. 단일 엔드 포인트. 원하는 것을 요청하고 받는 방식

  [GraphQL | A query language for your API](https://graphql.org/)

## Database

- PostgreSQL - 가장 인기 많은 RDBMS

  [PostgreSQL](https://www.postgresql.org/)

- SQLite - 하나의 파일만으로 동작하는 임베디드를 위한 RDBMS

  [SQLite Home Page](https://www.sqlite.org/index.html)

- Mongodb - 문서 지향 NoSQL

  [MongoDB: The Developer Data Platform](https://www.mongodb.com/)

- Redis: key-value 형태로 데이터를 저장하는 in-memory 기반의 NoSQL


## Web Application

- nestjs - nodejs 진영에서 가장 인기 많은 웹 프레임워크

[https://github.com/nestjs/nest](https://github.com/nestjs/nest)

- express - 오래 됨. 사용자 많음

- fastify - 빠름. 플러그인 다양

  [https://github.com/fastify/fastify](https://github.com/fastify/fastify)


## ORM

- Prisma - 인기 많음. 사용자 많음. ❗전체 텍스트 검색 이슈

  [https://github.com/prisma/prisma](https://github.com/prisma/prisma)

- Drizzle - SQL 구문과 유사하게 사용하고 가볍고 빠른 ORM

  [https://github.com/drizzle-team/drizzle-orm](https://github.com/drizzle-team/drizzle-orm)

## Web Framework

- Next - React 기반의 풀스택 웹 프레임워크

  [https://github.com/vercel/next.js](https://github.com/vercel/next.js)

- Nuxt - Vue 기반의 풀스택 웹 프레임워크

  [https://github.com/nuxt/nuxt](https://github.com/nuxt/nuxt)

- Remix - 사용자 경험을 중점으로 하는 풀스택 웹 프레임워크

  [https://github.com/remix-run/remix](https://github.com/remix-run/remix)

- SvelteKit - Svelte 기반의 강력하고 성능이 좋은 웹 프레임워크

  [https://github.com/sveltejs/kit](https://github.com/sveltejs/kit)

- Astro - 콘턴츠 중심 웹사이트를 위한 웹 프레임워크. 주요 UI 프레임워크와 통합

  [https://github.com/withastro/astro](https://github.com/withastro/astro)


## CSS

- Sass - CSS를 확장하여 여러 고급 기능을 제공. 빌드를 통해 CSS로 변환. 학습 곡선이 높음. 오래된 만큼 큰 커뮤니티

  [https://github.com/sass/sass](https://github.com/sass/sass)

- CSS Modules - 파일 별로 스타일을 캡슐화하여 고유 클래스명을 부여하는 방식. 컴포넌트 기반 개발에 적합

  [https://github.com/css-modules/css-modules](https://github.com/css-modules/css-modules)

- Styled Components - JavaScript 내에서 스타일을 정의하는 CSS-in-JS 라이브러리. 동적 스타일링에 유리. 오버헤드 존재

  [https://github.com/styled-components/styled-components](https://github.com/styled-components/styled-components)

- Tailwind CSS - 유틸리티 우선 방식의 CSS 프레임워크. 마크업하는 곳에서 스타일링을 함께 하여 빠른 개발. 커스터마이징 용이. 클래스 명을 고민하지 않아도 됨. 클래스 명이 너무 길어져서 지저분해 보임

  [https://github.com/tailwindlabs/tailwindcss](https://github.com/tailwindlabs/tailwindcss)


## Build Tools

- Vite - 웹 프로젝트를 위한 차세대 빌드 도구. `create-react-app`의 사실상 대체제
