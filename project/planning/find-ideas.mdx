## HMW(how might we)

‘어떻게 하면 ~할 수 있을까?’라는 질문을 통해 실행 가능한 해결책을 찾는 방법

- 어떻게 하면 사용자에게 회원가입을 쉽게 느끼게 할 수 있을까?
- 어떻게 하면 사용자가 쉽게 제품을 찾도록 할 수 있을까?
- 어떻게 하면 사용자가 결제를 쉽게 할 수 있을까?

## crazy 8

8분 안에 8개의 피처를 구상하는 방법

아이디어를 빠르게 구상하고 즉각적인 피드백

## 피처 우선순위 정하기

자원 대비 사용자의 가치를 고려

## As is To be

As is → 사용자 리서치를 통해 수집한 불편 및 니즈 

- (특정 사용자)가 (특정 행동을) 할 때 (이런 점이) 불편해요

To be → 사용자에게 가치를 제공할 수 있는 피처 정의
