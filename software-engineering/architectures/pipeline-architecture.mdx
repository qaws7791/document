## 파이프라인 아키텍처


파이프와 필터를 단순하고 작게 만들고, 이들을 조합하여 작업을 처리

```mermaid
graph LR
  필터 --파이프--> 필터2
  필터2 --파이프--> 필터3
  필터3 --파이프--> 필터4
  필터4 -.파이프.-> 필터5
```


## 파이프

- 한 소스에서 입력을 받아 다른 소스로 출력하는 역할
- 단방향, 점대점으로 구성
- 성능을 위해 파이프 간 페이로드를 적게 유지하는 것이 좋음


## 필터

- 자기 완비형
- 다른 필터와 독립적
- 일반적으로 무상태성
- 일반적으로 단일 작업을 수행
- 여러 필터를 결합하여 복잡한 작업을 수행할 수 있음


## 필터의 종류


### 프로듀서(producer)

- 프로세스의 시작점(source)
- 나가는 트래픽만 존재


### 변환기(transformer)

- 입력을 받아 일부 또는 전체 데이터를 반환하고 아웃바운드 파이프로 전달
- 함수형 프로그래밍의 `map`


### 테스터(tester)

- 입력을 받아 하나 이상의 기준에 대해 테스트, 결과에 따라 필요 시 결과를 생산
- 함수형 프로그래밍의 `reduce`


### 컨슈머(consumer)

- 파이프라인 마지막 지점
- 최종 결과를 데이터베이스에 저장하거나 화면에 표시하는 등의 작업을 수행
