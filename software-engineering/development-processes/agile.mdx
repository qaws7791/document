## 애자일

애자일은  Manifesto for Agile Software Development에서 정의한 4가지 핵심 가치와 12가지 원칙을 기반으로 하는 개발 방법론입니다. 애자일 방법론은 소프트웨어 개발 프로젝트를 진행할 때, 계획을 유연하게 조정하고, 작은 단위로 빠르게 개발하며, 고객의 피드백을 적극적으로 수용하는 방식으로 프로젝트를 진행합니다.

- 작동하는 소프트웨어
- 고객과의 협업
- 사용자의 반응에 대해 빠르게 대응
- 짧고 반복적인 스프린트 사이클
- 요구 사항 변경이 잦기 때문에 협업 중요

## 스프린트

스프린트(Sprint)는 애자일 방법론에서 사용하는 개발 주기를 의미합니다. 스프린트는 보통 1주일에서 1개월 정도의 기간으로 설정되며, 이 기간 동안 개발팀은 일정한 목표를 달성하기 위해 노력합니다. 스프린트는 다음과 같은 단계로 이루어집니다.


## MVP

MVP(Minimum Viable Product)는 최소 기능 제품을 의미합니다. MVP는 제품이 출시되기 전에 가장 중요한 기능만을 개발하여 사용자의 반응을 살펴보고, 제품의 방향성을 결정하는 데 사용됩니다. MVP를 개발하는 과정은 다음과 같습니다.

1. 아이디어 도출
2. 아이디어 검증
3. MVP 개발
4. MVP 테스트
5. MVP 개선

- MVP를 개발하고 지속적으로 피드백을 통해 개선
- 기능이 모자란 것이 아닌 니즈 검증을 위한 최소한의 기능을 먼저 개발
