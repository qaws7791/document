프로그래밍 패러다임은 프로그램을 작성하고 구조화하는 방식에 대한 철학적 접근 방식입니다. 각 패러다임은 문제를 해결하기 위한 다양한 방법과 스타일을 제공하며, 각기 다른 장단점과 응용 사례가 있습니다. 주요 프로그래밍 패러다임에는 다음과 같은 것들이 있습니다.


## 절차적 프로그래밍(Procedural Programming)

<Note>✨제어흐름의 직접적인 전환에 대해 규칙을 부과한다</Note>

- **특징**: 프로그램을 순차적으로 실행되는 절차(또는 함수)들의 집합으로 구성합니다.
- **언어**: C, Pascal, Fortran
- **장점**: 구조화된 코드 작성이 용이하고, 디버깅 및 테스트가 상대적으로 간단합니다.
- **단점**: 코드의 재사용성과 유지보수성이 떨어질 수 있습니다.

### 구조적 프로그래밍(structured programming)

절차적 프로그래밍의 하위 개념 중 하나. 무분별한 GOTO문에 대한 의존성을 줄여주는 데 유용하며, 이러한 점프를 if/then/else 와 do/while/until와 같이 더 익숙한 구조로 대체

## 객체지향 프로그래밍 (Object-Oriented Programming, OOP)

<Note>✨제어흐름의 간접적인 전환에 대해 규칙을 부과한다</Note>

- **특징**: 객체와 클래스 개념을 기반으로 프로그램을 작성합니다. 객체는 데이터(속성)와 이를 조작하는 함수(메서드)를 포함합니다.
- **언어**: Java, C++, Python, C#
- **장점**: 코드의 재사용성, 유지보수성, 확장성이 높습니다. 캡슐화, 상속, 다형성과 같은 개념을 통해 복잡성을 줄일 수 있습니다.
- **단점**: 비교적 복잡한 구조로 인해 배우기 어려울 수 있으며, 성능 상의 오버헤드가 발생할 수 있습니다.

## 함수형 프로그래밍 (Functional Programming)

<Note>✨할당문에 대해 규칙을 부과한다</Note>

- **특징**: 함수가 일급 시민으로 취급되며, 상태 변경과 부작용을 피하려는 경향이 있습니다. 불변성과 순수 함수를 강조합니다.
- **언어**: Haskell, Lisp, Scala, Erlang
- **장점**: 병렬 처리가 용이하고, 코드의 예측 가능성과 테스트 가능성이 높습니다.
- **단점**: 재귀와 고차 함수 같은 개념을 이해하고 사용하는 것이 어렵습니다. 일반적인 절차적 사고와는 다른 사고방식이 요구됩니다.

## 명령형 프로그래밍

<Note>프로그램의 상태와 상태를 변경시키는 구문의 관점에서 연산을 설명한다</Note>

- **특징**: 프로그램의 상태와 상태를 변경시키는 구문의 관점에서 연산을 설명한다
- **언어**: C, C++, Java, Python
- **장점**: 상태 변경과 부작용을 명확하게 관리할 수 있으며, 복잡한 상태 관리 로직을 효과적으로 처리할 수 있습니다.
- **단점**: 프로그램의 상태를 추적하고 관리하는 것이 어려울 수 있습니다. 실행 순서를 예측하기 어려울 수 있습니다.


## 선언형 프로그래밍

<Note> 실행 결과가 무엇과 같아야 하는지를 설명한다</Note>

- **특징**: 실행 결과가 무엇과 같아야 하는지를 설명한다
- **언어**: SQL, HTML, CSS
- **장점**: 문제 해결에 필요한 로직을 명확하게 표현할 수 있으며, 코드가 간결해집니다.
- **단점**: 실행 순서를 예측하기 어려울 수 있습니다.



## 논리형 프로그래밍 (Logic Programming)

<Note>✨논리적 문장으로 문제를 기술하고, 논리 추론을 통해 결과를 도출한다</Note>

- **특징**: 논리적 문장으로 문제를 기술하고, 논리 추론을 통해 결과를 도출합니다.
- **언어**: Prolog
- **장점**: 자연어 처리, 지식 표현 및 추론 같은 인공지능 분야에 적합합니다.
- **단점**: 다른 패러다임에 비해 범용성이 떨어지며, 복잡한 문제를 해결하기 어렵습니다.


## 선언형 프로그래밍 (Declarative Programming)

<Note>✨어떤 결과를 얻고자 하는지에 초점을 맞추며, 어떻게 할 것인지는 신경 쓰지 않는다</Note>

- **특징**: 무엇을 할 것인지에 초점을 맞추며, 어떻게 할 것인지는 신경 쓰지 않습니다.
- **언어**: SQL, HTML, CSS
- **장점**: 문제 해결에 필요한 로직을 명확하게 표현할 수 있으며, 코드가 간결해집니다.
- **단점**: 절차적인 제어가 필요할 때는 사용하기 어렵습니다.


## 이벤트 기반 프로그래밍 (Event-Driven Programming)

<Note>✨이벤트(사용자 입력, 센서 데이터 등)에 의해 프로그램의 흐름이 결정된다</Note>

- **특징**: 이벤트(사용자 입력, 센서 데이터 등)에 의해 프로그램의 흐름이 결정됩니다.
- **언어**: JavaScript, ActionScript
- **장점**: 사용자 인터페이스 개발에 적합하며, 비동기 처리에 강합니다.
- **단점**: 복잡한 이벤트 구조를 관리하기 어려울 수 있습니다.


## 병행 및 병렬 프로그래밍 (Concurrent and Parallel Programming)

<Note>✨여러 작업을 동시에 수행하여 성능을 높입니다</Note>

- **특징**: 여러 작업을 동시에 수행하여 성능을 높입니다.
- **언어**: Go, Rust, Erlang, Clojure
- **장점**: 멀티코어 프로세서의 성능을 최대한 활용할 수 있습니다.
- **단점**: 동기화 문제와 데이터 경합 문제를 해결하는 것이 어렵습니다.


## 메타 프로그래밍 (Metaprogramming)

<Note>✨프로그램이 다른 프로그램을 생성하거나 수정하는 프로그래밍 기법입니다</Note>

- **특징**: 프로그램이 다른 프로그램을 생성하거나 수정하는 프로그래밍 기법입니다.
- **언어**: Python, Ruby, Lisp
- **장점**: 코드의 유연성과 재사용성을 높일 수 있습니다.
- **단점**: 코드의 복잡성이 증가하고, 디버깅이 어려워질 수 있습니다.
