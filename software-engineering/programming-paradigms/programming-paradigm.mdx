# 프로그래밍 패러다임

## ➡️절차적 프로그래밍(Procedural Programming)

> [!NOTE]
> ✨제어흐름의 직접적인 전환에 대해 규칙을 부과한다

- **특징**: 프로그램을 순차적으로 실행되는 절차(또는 함수)들의 집합으로 구성합니다.
- **언어**: C, Pascal, Fortran
- **장점**: 구조화된 코드 작성이 용이하고, 디버깅 및 테스트가 상대적으로 간단합니다.
- **단점**: 코드의 재사용성과 유지보수성이 떨어질 수 있습니다.

### 구조적 프로그래밍(structured programming)

절차적 프로그래밍의 하위 개념 중 하나. 무분별한 GOTO문에 대한 의존성을 줄여주는 데 유용하며, 이러한 점프를 if/then/else 와 do/while/until와 같이 더 익숙한 구조로 대체

## 🐘객체지향 프로그래밍 (Object-Oriented Programming, OOP)

> [!NOTE]
> ✨제어흐름의 간접적인 전환에 대해 규칙을 부과한다

- **특징**: 객체와 클래스 개념을 기반으로 프로그램을 작성합니다. 객체는 데이터(속성)와 이를 조작하는 함수(메서드)를 포함합니다.
- **언어**: Java, C++, Python, C#
- **장점**: 코드의 재사용성, 유지보수성, 확장성이 높습니다. 캡슐화, 상속, 다형성과 같은 개념을 통해 복잡성을 줄일 수 있습니다.
- **단점**: 비교적 복잡한 구조로 인해 배우기 어려울 수 있으며, 성능 상의 오버헤드가 발생할 수 있습니다.

## 🟰함수형 프로그래밍 (Functional Programming)

> [!NOTE]
> ✨할당문에 대해 규칙을 부과한다

- **특징**: 함수가 일급 시민으로 취급되며, 상태 변경과 부작용을 피하려는 경향이 있습니다. 불변성과 순수 함수를 강조합니다.
- **언어**: Haskell, Lisp, Scala, Erlang
- **장점**: 병렬 처리가 용이하고, 코드의 예측 가능성과 테스트 가능성이 높습니다.
- **단점**: 재귀와 고차 함수 같은 개념을 이해하고 사용하는 것이 어렵습니다. 일반적인 절차적 사고와는 다른 사고방식이 요구됩니다.
